METHODOLOGY · 30,000-FT VIEW

The OSCP standalone box has the same shape every time: scan → enumerate every open service → find foothold → get shell → local privesc → root. Where people lose hours: skipping enumeration on ports they "know," giving up on a vector before cycling through obvious variants, and fighting one technique for 60 minutes when ten others would have finished it. This bank is organized as a decision tree, not a tutorial — use search to jump straight to the trigger you're looking at.

▸ 01
Enumerate every open port. Don't stop at HTTP because it looks juicy. The vuln is often on the boring port (SMTP/SNMP/rsync).
▸ 02
Run service-specific enum in parallel while you read web pages. nmap, gobuster, and smbclient can all be running while you click around the app.
▸ 03
Identify software + version → search exploit. Half of OSCP foothold is "version X has a public exploit." Don't manually fuzz before checking searchsploit / Google.
▸ 04
Foothold landed → run linpeas/winpeas immediately, but ALSO do manual sudo -l / whoami /priv / cron / SUID checks. Auto-tools miss things.
▸ 05
Found creds anywhere? Spray them everywhere. Same password on different users, different services. PG boxes reuse creds constantly.
▸ 06
Stuck for 30 min on one path? Pivot to another vector or another box. Come back with fresh eyes. Don't grind.
▸ 07
Save MSF for one box. Manual exploitation everywhere else. If a box screams "MSF target" (BoF, ms17-010, helpdesk-style java) — that's the one.
▸ 08
Grab local.txt before privesc, proof.txt after. Take screenshots immediately. Don't lose flags to a session crash.

[ ⚠ ] ANTI-PATTERNS · DON'T DO THIS
Skip a service because port "looks boring" (rsync, finger, snmp)
PG Fail = rsync. PG ClamAV = SNMP. PG Bratarina = SMTP. The boring port IS the vuln.
Manual fuzz before searching the version
whatweb / banner → searchsploit → google. Half of OSCP foothold is "version X has public RCE."
Use MSF on box #1 because it's "easy"
you only get ONE MSF use on the standalones. Save it for a BoF box (Kevin/Kyoto) or ms17-010 (Internal).
Run linpeas and stop there
linpeas misses creds in custom paths. Always also: sudo -l, manual cron walk, find / -perm -4000, /var/mail, .bash_history.
Forget to su as every other user with that password
PG boxes (Apex, codo, Boolean) reuse passwords across accounts. Spray any cred you find against every shell user.
Run BoF without setting up rlwrap + writing the bytes file
BoFs need persistent socket, hex-precise payload, and patience. Use the OSCP-provided BoF template — don't improvise.
Grind one vuln for 60 min when nothing's working
30 min cap per vector. Move to the next. Come back fresh.
Drop SharpHound / loud tools on a Linux box "just in case"
Linux boxes don't run AD. Use linpeas / pspy / manual.
Skip /etc/passwd before privesc — "I'm not root yet"
readable /etc/passwd reveals shell-having users. /etc/shadow readable = unshadow + crack. Always check.
Submit proof.txt without a screenshot showing IP + flag together
graders need: ip a + cat proof.txt in the same screenshot. Take it the moment you root.
Restart the VM when stuck
re-revert wastes 5 min. Re-enumerate first — you almost certainly missed something.
Forget to grab local.txt before going for root
always: shell → enum → local.txt → privesc → proof.txt. local.txt is points already earned.
