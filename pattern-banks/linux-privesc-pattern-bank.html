The discipline: every box gets the full sweep before any exploit attempt. linpeas runs first in the background while you go through the manual list. Re-enumerate after every lateral hop — same machine, new user, completely different attack surface.

▸ 0. CONTEXT
  • Full TTY — python3 -c "__import__('pty').spawn('/bin/bash')"
  • Terminal size — Ctrl+Z, stty raw -echo; fg, export TERM=xterm
  • Confirm user — id; whoami; hostname; uname -a
  • Distro + version — cat /etc/os-release; uname -r
▸ 1. AUTOMATED
  • linpeas — ./linpeas.sh -a -P PWD in background
  • LES — ./les.sh
  • pspy64 — passive process watch (catches root cron)
▸ 2. SUDO
  • sudo -l blank pwd AND user pwd
  • GTFOBins for every entry
  • NOPASSWD first · env_keep LD_PRELOAD/LD_LIBRARY_PATH
  • Sudo version — sudo --version → CVE-2021-3156 if <1.9.5p2
  • CVE-2019-14287 — sudo -u#-1 trick
▸ 3. SUID / SGID
  • find / -perm -4000 -type f 2>/dev/null
  • find / -perm -2000 -type f 2>/dev/null
  • GTFOBins for each known binary
  • Unknown bin — strings, ltrace, strace -f -e openat
  • Relative paths → PATH hijack
▸ 4. CAPABILITIES
  • getcap -r / 2>/dev/null
  • cap_setuid+ep → GTFOBins (python, perl, ruby)
  • cap_dac_read_search → read /etc/shadow
▸ 5. CRON / TIMERS / SERVICES
  • cat /etc/crontab; ls -la /etc/cron.*
  • systemctl list-timers --all
  • Every script in cron → check writable
  • pspy for dynamic / out-of-crontab
  • PATH abuse — relative paths in cron script
  • Wildcard expansion — tar/chown/rsync *
  • Writable .service units
▸ 6. GROUPS
  • id — what groups?
  • docker → mount /
  • lxd → image import privesc
  • disk → debugfs raw fs
  • adm → read /var/log
▸ 7. WRITABLES
  • ls -la /etc/passwd /etc/shadow /etc/sudoers
  • SSH authorized_keys for other users
  • find / -writable -type d 2>/dev/null | grep -v proc
▸ 8. CRED HUNT
  • /var/www config files (DB creds)
  • /opt custom apps with hardcoded creds
  • ~/.bash_history · ~/.mysql_history · ~/.viminfo
  • /var/mail/<user> (PG Plum, Fired)
  • grep -ri 'password' /etc/ 2>/dev/null
▸ 9. PASSWORD REUSE
  • su every user with current pwd
  • Try usernames as passwords
  • Any cracked hash → try across users
▸ 10. KERNEL (LAST RESORT)
  • LES output
  • Polkit pkexec (CVE-2021-4034)
  • DirtyPipe (5.8-5.17)
  • OverlayFS Ubuntu (CVE-2021-3493)

[ ⚠ ] ANTI-PATTERNS
linpeas crashes → "guess I will skip enum"
Run the manual checklist by hand: sudo -l, find / -perm -4000, getcap -r /, cat /etc/crontab, id, ls -la /etc/passwd. 10 minutes, no excuses.
Kernel exploit on first try → kernel panic → reset box
Kernel exploits are LAST resort. Sudo, SUID, cron, groups, writables — all of those first. PG/exam boxes intentionally include misconfigs to avoid kernel exploits.
sudo -l prompts for password → assume no sudo
Try the user pwd if you have it. Try blank. Try common reuse pwds from elsewhere. Many boxes need creds discovery before sudo -l reveals anything.
SUID found → GTFOBins has no entry → give up
Unknown SUID = potential goldmine. strings <bin>, ltrace <bin>, strace -f -e openat <bin>. Look for relative paths (PATH hijack) or system() calls.
Edit cron script, wait 5 min, nothing happens
Check the schedule. * * * * * = every min. /15 * * * * = xx:00/15/30/45. Also: did you make it executable?
Got root once via kernel exploit, try same on every box
Kernel exploits are version-specific. Always check uname -r + cat /etc/os-release first. Wrong kernel = panic.
Skip /etc/passwd check because "obviously not writable"
PG Fail proved it. ls -la /etc/passwd takes 1 second. If writable: openssl passwd → root2 line → su. Easiest privesc on the planet.
Find creds in config file, no clear use → ignore
Every cred goes through su <every user> first. People reuse passwords for everything. PG Apex / Plum / Fired all hinged on this.
"Box only has SSH and HTTP, must be web exploitation"
Once you are in via web, the privesc surface is the WHOLE machine. Run pspy. Check 127.0.0.1 ports. Check service account.
pspy shows root cron but you cannot write the script
Check the PATH usage. Check writable dir with wildcards. Check sourced files. Indirect write paths are how PG Muddy worked.
